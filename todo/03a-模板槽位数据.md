# 模板槽位数据（Block Template Slot Data）

> `03-地块模板设计.md` 的**配套数据文档**。03 定义了 `TemplateSlot`/`LocationTemplate` 的**设计**（hotspot、kind、choices、Footprint），但缺**逐模板的槽位表**与**建筑码→场景映射**——本文把这些具体数字补齐，让"第一个村庄"可直接落地。
>
> 调研来源：`origin/others/文档/城镇生成逻辑.md` §6（`generate_location` 槽位表）、§7（建筑码→建筑对象）、§7.1（随机物体码→车辆）；§2/§5 类型码→模板号（已在 03 §1 汇总，本文仅交叉引用）。
> 坐标约定：偏移 `dx/dy` 为**像素绝对偏移**，原版 `generate_location` 里 `x = hotspot + dx`，**1:1 直接照搬**（tile=60px 已使尺度对齐，见 03 §附 / 01 尺度规格）。
> 前置依赖：建筑场景 `environment/building/*.tscn`（自包含）、环境物体场景 `environment/objects/*.tscn`（见 `02-环境物体场景.md`）。

## 1. 字段映射：原版 `t518` 行 → Godot `TemplateSlot`

原版每个槽位是 `t518` 的一行 `[kind, code, x, y]`；本项目用 `environment/location_template/template_slot.gd`（`Marker2D`）承载，字段一一对应：

| 原版字段 | 含义 | → Godot `TemplateSlot` | 说明 |
|---|---|---|---|
| `kind`（col 0） | 元素种类 7/6/1/3/4 | `type: Type` 枚举 | 7→BUILDING、6→VEHICLE、1→TREE、3→PROP、4→CONTAINER |
| `code`（col 1） | 资源整数码，可 `choose`/`random` | `choices: Array[PackedScene]` | **整数映射表整张消失**：直接引用 `.tscn`。1 个=固定，多个=随机（对应原版 `choose`） |
| `x = hotspot + dx` | 世界 X | 节点 `position.x = dx` | 节点自身 position 即相对 hotspot 的偏移（父节点 `LocationTemplate` 摆在 hotspot） |
| `y = hotspot + dy` | 世界 Y | 节点 `position.y = dy` | 同上 |
| `dx = +A+random(B)` | 位置随机（路边车/树） | `position_jitter: Vector2` | 节点 position 取**中心值**，jitter 取**半幅**（见 §6 车辆换算） |

> 枚举名以代码为准：`TemplateSlot.Type { BUILDING, VEHICLE, TREE, PROP, CONTAINER }`。`LocationTemplate.build()` 已实现"遍历 Slots → `slot.pick(rng).instantiate()` → `position = slot.spawn_position(rng)`"，本文的表只需照填即可被消费。

## 9. 裁决 03 §8 的 5 个待定决策

> 结合本文数据给出**具体建议**，目标是用最短路径打通"第一个村庄"（03 里程碑 C1~C4）。

| 决策 | 03 原议题 | 本文建议（决策） | 理由（与本表一致） |
|---|---|---|---|
| **D1 槽位形态** | 单一 `TemplateSlot`+kind 枚举，还是每类一个子类 | **已定：单一 `TemplateSlot` + `Type` 枚举** | 代码已如此实现（`template_slot.gd`）。本文所有表都只用 `type`/`choices`/`jitter` 三字段即可表达建筑/车/树/特殊，无需子类。 |
| **D2 道路模板范围** | 本轮做不做道路地块的车/树/容器 | **本轮不做道路模板对象**，只做村庄 9 的建筑 | 道路 kind=4/5 内容（十字 93 个"容器"）原文档未澄清（03 §2 待查），且车主力在路边但 dx/dy 未逐项给出；先把村庄建筑跑通，道路对象留下一轮。 |
| **D3 共享 SlotPool** | 现在就上 `.tres` 池还是先 inline | **先 inline `choices`** | 首版仅村庄 9，车/树池各只被复用 ~3/9 次，inline 数组够用；待城市/军营/医院全上（车槽 ×几十）再抽 `all_vehicles.tres` / `all_trees.tres`（02 §3 / 03 §7.1）。 |
| **D4 地面足迹画法** | `TileMapLayer` 可视化画 vs `Rect2i` 列表照搬 | **`Footprint` TileMapLayer 可视化画** | `location_template.gd` 的 `_collect_footprint` 已按 `footprint.get_used_cells()` 实现，与可视化画法直接吻合；照搬 `Rect2i` 需另写转换，无收益。 |

## 10. 如何把这张表填进 `TemplateSlot` 场景

以村庄 9 为例（对应 §3.1）：

1. **新建/打开模板场景**：`environment/location_template/` 下复制 `location_template.tscn` 为 `village_9.tscn`，根节点 `LocationTemplate`，设 `category = VILLAGE`。
2. **在 `Slots` 节点下加 11 个 `TemplateSlot`**（实例化 `template_slot.tscn` 作为子节点）。
3. **逐个设置每个槽位**（按 §3.1 表行）：
   - `type` = `BUILDING`；
   - `position` = `(dx, dy)`（如 #1 填 `(844, 91)`）——节点 position 即相对 hotspot 偏移，编辑器画布可拖拽微调，所见即所得；
   - `choices` = 拖入对应 `.tscn`（如 #1 拖 `building/village_brown.tscn` … `hostel.tscn` 共 5 个；车库槽拖 `garage_blue/green`；棚/超市槽拖 `shed/supermarket`）；
   - `position_jitter` = `(0,0)`（建筑不抖）。
4. **画 Footprint**（可选，D4 已定可视化）：在 `Footprint` TileMapLayer 上画村内小路/路口的路面格；运行时 `build()` 会并入全局 terrain（`location_template.gd` 已实现）。
5. **注册到 ground.gd**：把 `village_9.tscn` 加进村庄类型的模板集（`REGISTRY[VILLAGE]`，对应 03 §5.3），让世界生成时按类型 `choose` 到它。
6. **车/树槽（第二轮）**：加 `type=VEHICLE/TREE` 的槽，`choices` 填 §6 车池 / §7 树池；车槽设 `position`=区间中心、`position_jitter`=半幅（§6 换算）。

> 校验：`TemplateSlot` 自带 `_get_configuration_warnings`（choices 空会警告）与 `_draw`（按 type 上色、画 jitter 范围框），编辑器里能直接看到每个槽的种类与抖动范围，便于核对本表。
