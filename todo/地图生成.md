## 尺度规格（已确认，后续阶段的基准）

本项目与原版**世界尺度 1:1**，关键换算：

| 概念 | 本项目 | 原版 | 说明 |
|---|---|---|---|
| 1 个 block（地块） | **17×17 tile = 1020×1020 px** | 地点间距 1020px | `17 tile × 60px = 1020px`，正好对上 |
| 整张地图 | 16×16 block = 272×272 tile = **16320px** | 16×1020 = 16320px | 完全一致 |
| terrain 层 tile | **60px** → 每 block 17×17 | — | `ground_terrain.tres` |
| decoration 层 tile | **30px** → 每 block 34×34 | — | `ground_decoration.tres` |
| 城镇内容区 | ≈16.67 tile（1000px） | 建筑偏移 0~1000px | 比 block 小一圈，留 ~20px 缝隙 |

**两个易混的数**：`1020px` 是**地块间距**（= block 大小 = 17 tile）；`1000px` 是**城镇内容区**（建筑铺满范围，≈16.67 tile，非整数正常）。

**建筑摆放规则**：用**像素级绝对坐标**，与 17×17 tile 网格无关——
`block 世界原点 = block坐标 × 1020`，建筑放在 `原点 + (原版 dx, dy)`，**原版偏移 1:1 直接照搬，无需缩放**（因 tile=60px 已使尺度对齐）。
→ 17×17 tile 网格**只用于地面渲染**（草/路/水，阶段 B）；建筑定位是自由像素坐标（阶段 C）。

> ⚠️ 前提：terrain tile 必须保持 **60px**。若改 tile 尺寸，17 tile ≠ 1020px，城镇会被压缩/拉伸，需重新换算建筑偏移。

## 阶段 A — 地点类型分类（地基，最优先）

**核心是「地点要分类型」**（村庄/城市/医院…），因为阶段 C 要据此决定生成哪些建筑——这与加载策略无关，是「造什么」的问题，**必做**。

数据结构用 **`grid[16][16]`**（16×16 二维数组，每格存一个 `BlockType`，即「该地块是什么」）。因为要「完全和原版一致」，而原版的十字路口、秘密地点、加油站等逻辑全是在这张网格上「扫描相邻格」完成的——用网格能直接照搬原版算法，最省事、最不易错；用 town 列表则要反复遍历查找坐标，慢且易错。类型用 `BlockType` 枚举，别用原版魔法数字(9/6/13…)。

- [x] **A1** 定义 `BlockType` 枚举（地点 5 类 + 空地 + 道路方向/十字 + 秘密地点等）+ `grid`（16×16，每格一个 `BlockType`），作为生成与渲染的统一数据源
- [x] **A2** 地点分 5 类落子（**完全照原版**）：村庄/城市/军营/医院/消防站，按关卡数量表（关卡0：20/3/0/1/1）用现有 3×3 算法落子并写入 `grid`
- [x] **A3** roads_2 补 **十字(14)** 显式标记（横纵路交叉处）
- [x] **A4** 补 **秘密地点(2)**：扫描「周围一圈空 + 坐标在 2~15」候选格，随机选一个
- [x] **A5** 每个地点记录 hotspot 世界坐标（供阶段 C 实例化用）

> 地点类型**完全照原版分 5 类**（已定），落子时直接赋值进 `grid`。
