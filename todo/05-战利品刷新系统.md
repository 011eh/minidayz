## 2. 数据模型：`LootTable` Resource（.tres）

把 21 池做成**一个 Resource**，而非散在脚本里的硬编码字典——便于在编辑器查看/微调、热重载、被 autoload 预加载一次。

```gdscript
# loot/loot_table.gd
@tool
class_name LootTable extends Resource

# 键 = loot_type(1..21)，值 = 该池的物品 ID 列表（含重复=加权）
@export var pools: Dictionary[int, PackedInt32Array] = {}

# 从某池加权抽 1 个物品 ID（重复条目天然成为权重）
func roll(loot_type: int) -> int:
    var pool: PackedInt32Array = pools.get(loot_type, PackedInt32Array())
    if pool.is_empty():
        return -1
    return pool[randi() % pool.size()]
```

- **加权 = 重复条目**：原版 `choose(1,1,2,3)` 让 1 出现两次即 2 倍权重。直接把它存成 `[1,1,2,3]`，抽取就是 `pool[randi() % size]`——**无需另建权重表**，与原版语义 1:1。这是把「`choose` 重复」翻成 Godot 加权随机的最忠实做法。
- 资源文件落在 `loot/loot_tables.tres`，数据来自 `parsed/loot_tables.json`（一次性导入脚本/手填皆可，见 §7 P0-1）。
- `PackedInt32Array` 比 `Array[int]` 省内存、序列化紧凑；池规模 1..56 完全够用。
- 单例引用：autoload `LootSystem`（§3）`@export var table: LootTable` 指向该 .tres，全局只此一份。

## 3. Godot 节点结构

### `LootPoint`（沿用 `02` 文 §3 提案，本文完工）

```gdscript
# loot/loot_point.gd
@tool
class_name LootPoint extends Marker2D

@export var loot_type: int = 1            # 1..21，对应原版 iv[0]
@export var spawn_radius: float = 24.0    # 在锚点附近的散布半径
@export var auto_register: bool = true    # _ready 时自动登记到 LootSystem

var _spawned: Node = null                 # 当前已刷出的地面 Item（被回收后置空）

func _ready() -> void:
    if Engine.is_editor_hint():
        return
    if auto_register:
        LootSystem.register_point(self)

# 由 LootSystem 调用：清掉旧的、按池抽 1 件、在附近落地
func respawn() -> void: ...
```

- 物体场景里**只摆锚点**（车辆 `LootAnchors/`、容器派生场景内），`auto_register` 让它自报到全局系统。
- 锚点自身不渲染（Marker2D 编辑器图标即可）；运行时由 `LootSystem` 统一驱动 `respawn()`。

### autoload `LootSystem`（核心调度）

├── RespawnTimer  Timer                # wait_time=1480, autostart, one_shot=false
└── (运行时维护)
    points: Array[LootPoint]           # 所有已登记锚点

## 4. LootPoint 出货流程（`respawn()`）

```gdscript
func respawn() -> void:
    if is_instance_valid(_spawned):     # 上一件还在 → 先清掉（重刷=焕然一新）
        _spawned.queue_free()
        _spawned = null
    var id := LootSystem.table.roll(loot_type)
    if id < 0:
        return
    var it := ItemCreator.create_item_from_id(id)   # 见 §8 前置：手枪段必须先补
    if it == null:
        return
    get_tree().current_scene.add_child(it)          # 挂进世界（非玩家背包）
    var off := Vector2.RIGHT.rotated(randf() * TAU) * randf() * spawn_radius
    it.global_position = global_position + off
    _spawned = it
    LootSystem.register_ground_item(it)
```

- 与「加进玩家背包」区分：背包路径是 `player.inventory.put_to_inventory(it, false)`；地面刷新走 `add_child` + 设坐标，让其 `InteractionArea` 被玩家探测后拾取。
- `spawn_radius` 让同一容器多锚点出货不叠在一点。

## 5. 刷新/回收循环（Godot 方式，不照搬原版逐帧距离扫描）

原版每帧扫所有 loot 点/掉落算距离（HTML5 平台所迫）。本项目**逻辑刷新交给单个 Timer，渲染剔除交给引擎**，与 `01-地图生成.md` 阶段 D 立场一致。

- **周期**：`RespawnTimer` 每 **1480s** `timeout` 一次，触发一轮全局刷新。这是「逻辑级」批处理，每 25 分钟一次的扫描成本可忽略，无需逐帧。
- **重刷条件**：遍历 `points`，对**距最近玩家 > 1000px** 的锚点调 `respawn()`（避免在玩家眼前刷）。单人模式「最近玩家」=唯一玩家。
- **回收旧物**：遍历 `ground_items`，**距玩家 ≥ 1500px** 的 `queue_free()` 并移出列表，腾位给新刷（对齐原版「旧掉落距玩家 ≥1500 销毁」）。回收**先于**重刷执行（原版同序：先腾位再刷）。
  - 原版还排除「与帐篷重叠 / 保护箱 `iv[2]=88`」的掉落——本项目帐篷/保护箱系统未立项，**先不实现该豁免**，待帐篷系统时补条件（标 ⏸️）。
- **可见性/性能**：锚点与地面物**全量常驻场景树**，不做按距离 create/erase。屏幕外渲染由引擎自动剔除；如需进一步省 CPU，沿用 `Building` 的 `VisibleOnScreenEnabler2D` 门控思路，但 loot 锚点本身无 `_process`、`Item` 探测在玩家侧，通常无需额外门控。
- **区域级回收 `respawn_radius2=2000`**：原版用 `t507` 区域刷新点重置 `t518` 格子计数——这是 HTML5 分块管理的产物，本项目**不实现**，由「每锚点独立 1480s 重刷 + 1500px 回收」等效覆盖（与 `01` 阶段 D 决定一致）。

> 多人/分屏暂不在范围；`_get_player()` 返回单个玩家即可，接口预留「最近玩家」语义以便日后扩展。

- 与普通 loot 点区别：弹药盒**充能而非重刷物品**，单独遍历列表处理。

## 7. 空投战利品表（本系统持有数据，事件 spawn 在别处）

空投箱 `t704` 的**生成/落地/冷却**属事件系统（`01-地图生成.md` 阶段 E）；本文只持有**开箱时的分层 loot 表与抽取逻辑**，供事件系统在玩家碰箱时调用。

```gdscript
# loot/airdrop_table.gd —— 由空投事件在开箱时调用
# 源：空投战利品概率表.md §2
static func roll_airdrop(difficulty: int) -> int:
    if [1, 2].pick_random() == 1:              # 50% 基础食物
        return [1,2,3,8,9,30,31,32,33].pick_random()
    var mid_thr := [40, 39, 38][difficulty]    # N/R/V
    if randi() % 50 < mid_thr:
        return MID_POOL.pick_random()          # 中级池(20项)
    var high_thr := [10, 9, 8][difficulty]
    if randi() % 11 < high_thr:
        return HIGH_POOL.pick_random()         # 高级池(22项)
    return TOP_POOL.pick_random()              # 顶级池(28项)
```

- 嵌套 Else 结构与原版一致：实际顶级出率 = `0.5 × (1-中级率) × (1-高级率)`，难度越高越易掉到稀有层。
- 三池明细见 `空投战利品概率表.md` §2（中/高/顶级），此处只放结构，不复制全列表。
- **定向空投（SEED_TD=1）** 用固定组合而非分层（`空投战利品概率表.md` §3），且涉及对象号→武器 ID 映射尚有 ◻️ 待核实——**本轮不实现**，标 ⏸️。

## 10. 决策

### 已定 ✅
- **数据用 Resource**：21 池存 `LootTable`(.tres) 的 `Dictionary[int, PackedInt32Array]`，**重复条目即权重**，抽取 `pool[randi()%size]`——忠实翻译原版 `choose`，无独立权重表。
- **刷新用 autoload + 单 Timer**：`LootSystem` + `RespawnTimer(1480s)`，每轮「回收远处旧物(≥1500px) → 重刷远处锚点(>1000px) → 弹药盒充能(≥500px)」；**不照搬原版逐帧距离扫描，也不做按距离 create/erase**（对齐 `01` 阶段 D / `02` §7）。
- **地面物复用 `Item` 场景**：不新增地面物类型，刷新系统只创建/摆放/回收 `item/item.tscn`，拾取走既有 `detection_area` 链。
- **职责切分**：本文持有 **loot 数据表（建筑池 + 空投分层）**；空投箱**生成/冷却/落地**归事件系统（`01` 阶段 E），不在此实现。
- **可搜刮首开一次性出货**，后续焕新由 1480s 周期接管。

### 待定 ❓
- [ ] **D-手枪边界**：`Pistol` 脚本的 ID 区间确切边界（190..201 还是 190..202？），定 §8 F-1 的 `range`。
- [ ] **D-木材 ID**：砍树掉落的木堆/木棍具体物品 ID（需查物品表确认）。
- [ ] **D-空投难度入口**：`difficulty`(0/1/2 = N/R/V) 从哪取（场景开局设定？全局难度单例？）——与事件系统对齐时定。
- [ ] **D-`loot_type` 赋值源**：建筑类型→`LootPoint.loot_type` 的精确映射，权威值在城镇/建筑模板写入处（`建筑容器战利品表.md` §待核实 1），随 `01` 阶段 C 落模板时确定，本系统只消费。
