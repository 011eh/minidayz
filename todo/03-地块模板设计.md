## 0. 核心认知（先纠正三个误区）

1. **不只是「城镇」模板，而是「地块」模板**。原版每个非空格子都由一个**模板号**展开——村庄/城市/军营/医院/消防是模板，**道路（纵/横/十字）同样是模板**，野外森林也是（但走程序散树）。命名用「地块模板」更准确。
2. **车辆不只在城镇，主要在路边**。三种道路模板都铺大量车（十字 13 辆、纵路 10 辆、横路 4 辆），地点模板里也有（医院最多 11 辆）。详见 §2。
3. **模板不止「摆对象」，还「铺地面``」**。每个地块除了对象槽位，还会往地面地形层盖一块「路面足迹」（医院是大铺装、村庄是连接房子的小路）。详见 §3。

## 1. 原版架构：一个「模板号」驱动两个函数

原版**没有单一的「模板对象」**。一个地块的全部内容，由它的模板号（`t507.iv[0]`）在**两个不同函数**里分别展开：

| 函数 | 阶段 | 按模板号产出 | 内容 |
|---|---|---|---|
| `generate_location(模板号)` | 步骤6 填数据 | **对象槽位** → `t518` 数组 | 建筑/车/树/容器/特殊 |
| `create_map_location(模板号)` | 步骤8 实例化 | **① 地面足迹** → `t945` 地形层<br>**② 装饰散布** → `t221` 装饰层 | 路面铺装 / 草地杂物 |

> 即：模板号是个抽象编号，同时管「摆什么对象」「铺什么地面」「撒什么装饰」三件事，但代码上分散在不同函数。

**类型码 → 模板号**（步骤5 `generate_array_tomap`，含随机）：

| 地块类型 | → 模板号 | 备注 |
|---|---|---|
| 村庄 | `choose(9, 4, 16, 17)` | 4 套排列 |
| 城市 | `choose(6, 21)` | 2 套 |
| 军营 | `22`（部分后处理升级为 `5`） | |
| 医院 | `11` | |
| 消防站 | `15` | |
| 道路 横/纵/十字 | `13 / 12 / 14` | 道路也是模板 |
| 空地/森林 | `choose(3,1,2,8,10,…)` | 走程序散树，非固定槽位 |
| 秘密地点 | `choose(23..49)` | 16 种特殊 |

## 2. 模板元素之一：对象槽位

对应原版 `t518` 每行 `[kind, code, dx, dy]`。每个槽位 4 个字段：

| 字段 | 含义 | 例子 |
|---|---|---|
| `kind` | 元素种类 | 7=建筑、6=车辆、1=树、3=特殊(信箱/牌子)、4=容器 |
| `code` | 资源编号，**可固定可随机** | `9` 固定 / `choose(9,10,11,12,13)` 随机色 |
| `dx, dy` | 相对 hotspot 的**像素**偏移 | `+844, +91` |

- 部分槽位**位置也带随机**：车辆 `dx = +100 + random(300)`（路边车散开）。
- **野外森林是例外**：不是固定槽位，`Repeat(16)` 在 4×4 子格程序化散树。

## 3. 模板元素之二：地面足迹（新发现，原版真实存在）

原版有一张地面地形 tilemap **`t945`**（贴图 `ground_lvl{1-4}_tilemap.png`）。`create_map_location` 阶段，**每个地点和道路都往这张图盖一块路面足迹**：

1. **填实心（tile 15）**：用 `SetTileRange` 把足迹区域刷成 tile 15（路面满铺）。形状各模板不同：
   - 医院：7 个矩形拼 L/十字形大铺装；
   - 村庄 9：13 个矩形 = 村内多条窄路（连接各房子的小径）+ 外接路口；
   - 道路：路条 + 路口。
2. **8 邻居自动地形**：扫每格，按 8 邻居是否为 tile 15（`CompareTileAt == 15`）换成边/角过渡 tile（`SetTile 0~12`），让路面与草地平滑衔接。

> **本质 = Godot `set_cells_terrain_connect`**：原版「填 15 + 邻居 autotile」就是 Godot 地形连接做的事。
> 道路和地点铺的是**同一个路面 terrain**，所以村内小路能无缝接到村外的路。
> 坐标系：足迹用 block 内 tile 偏移（17×17 格），与本项目 `BLOCK_SIZE_IN_TILE=17`、terrain 60px **完全一致，偏移 1:1 直接用**。

**现状**：本项目阶段 B 已用 `set_cells_terrain_connect(road_cells, 0, 1, …)` 给**道路**做了同样的自动地形；缺的是把**地点足迹**也并入这个 terrain 填充。

## 4. 建筑自身：多层自包含（不归模板管）

一栋建筑不是单图，是**多层精灵叠加**（以教堂 code 16 为例，`create_map_element` kind=7）：

```
教堂对象 = b_church_int(底座/地板, Buildings_base 层)   ← 建筑脚下那块铺装，属于建筑自己
         + b_church(主体外观)
         + 墙体碰撞(按精灵图点 w1~w4 定尺寸)
         + 阴影(Buildings_shadows)
         + 僵尸刷新点 ×N + 内部/loot
```

→ **建筑脚下的小块铺装 = 建筑自带的底座层**，与模板无关。模板只负责把整栋 `building.tscn` 摆到槽位。
→ 对应 Godot：每个 `building.tscn` 自己包含 `BaseFloor / Body / Walls / Shadow / Interior / ZombieSpawns / Loot`。这是阶段 C 的前置依赖（**建筑场景需自包含其内部**，否则城镇是空壳）。

## 5. Godot 场景化设计

把原版分散在两个函数的内容，**收进一个模板场景**（比原版整洁，且可视化）。

### 5.1 槽位节点 `TemplateSlot`（原子）

`@tool`，`extends Marker2D`（编辑器显示十字，位置即节点 `position`，画布可拖）：

```gdscript
@tool
class_name TemplateSlot extends Marker2D
enum Kind { BUILDING, VEHICLE, TREE, PROP, CONTAINER }
@export var kind: Kind = Kind.BUILDING
@export var choices: Array[PackedScene] = []        # 候选场景；1个=固定，多个=随机 choose
@export var position_jitter: Vector2 = Vector2.ZERO # 位置抖动范围(路边车/树用)
# 位置 = 节点自身 position（编辑器画布拖拽，所见即所得）
# 可选：@tool _draw() 画出 choices[0] 贴图，摆位时看得见建筑轮廓
```

- **直接存 `PackedScene` → 原版 code→资源映射表整张消失**（Construct2 用整数 code 是平台所迫，Godot 直接引用场景）。

### 5.2 模板根 `BlockTemplate`（容器 + 生成）

`.tscn` 根节点，`@tool extends Node2D`：

```
BlockTemplate (Node2D, 根, 挂 block_template.gd, @export category: BlockType)
├── Footprint    (TileMapLayer)   ← 地面足迹：编辑器画出路面形状 【对应原版 t945 分支】
├── Slots        (Node2D 分组)
│   ├── TemplateSlot kind=BUILDING choices=[brown,red,green,yellow,hostel] pos=(844,91)
│   ├── TemplateSlot kind=VEHICLE  choices=[<车池>] jitter=(300,0)         pos=(635,195)
│   └── TemplateSlot kind=TREE     choices=[<树池>] …
├── CityGround    (Sprite2D)       ← 仅城市模板：整块沥青底图(原版 city 模板单独铺)
└── VisibleOnScreenEnabler2D       ← 可选(阶段D)：矩形≈1020×1020，进屏开/离屏关本town的process
```

核心方法 `func build(rng) -> void`：
1. 把 `Footprint` 的 `get_used_cells()` 偏移 hotspot 后，交给应用器并入全局 terrain；
2. 遍历 `Slots` 下每个 `TemplateSlot`，`choose(choices)` 选一个 `instantiate()`，叠加 jitter 位置，`add_child`；
3. 释放槽位 marker（`queue_free()`）。

> 实例化后**整个地块就是这一个节点的子树**，天然满足「按 town 粒度管理/剔除」（阶段 D）。

### 5.3 应用器（`ground.gd`）

```gdscript
var pavement_cells := {}
for loc in all_cells:                      # 遍历所有非空格子(含道路)
    if loc.type == 森林类: scatter_trees(loc); continue   # 野外走程序散树
    var tpl: PackedScene = choose(REGISTRY[loc.type])      # 类型→模板集 随机选
    var block := tpl.instantiate()
    block.position = loc.hotspot           # hotspot = block坐标 × 1020px
    add_child(block)
    block.build(rng, pavement_cells)       # 收集足迹 cells + 实例化对象
# 全部地块的足迹统一一次性 terrain-connect，和道路共用同一路面 terrain
ground_layer.set_cells_terrain_connect(pavement_cells.keys() + road_cells, 0, 1, false)
```

## 6. 边界划分（场景内 vs ground.gd）

| 原版「模板号产出」 | 放哪 | 理由 |
|---|---|---|
| 对象槽位（建筑/车/树/容器） | ✅ **模板场景**（`TemplateSlot`） | 固定布局，要可视化摆位 |
| 地面足迹（路面铺装形状） | ✅ **模板场景**（`Footprint` TileMapLayer） | 固定形状，画一次，应用时并入全局 terrain-connect |
| 城市整块底图 | ✅ **模板场景**（`CityGround` Sprite2D） | 城市特例 |
| 建筑内部（底座/墙/影/僵尸/loot） | 🔵 **building.tscn 自包含** | 与模板解耦 |
| 装饰散布（t221 草地杂物） | 🟡 **留 `ground.gd`**（已有 `render_decoration`） | 随机散布、非固定形状，按 block 类型撒即可 |
| 类型→模板号 choose、遍历落地、rng | 🟡 **`ground.gd` + 注册表** | 属于世界生成，与单模板解耦 |

> 原则：**「确定的东西」（对象位置、路面形状）进模板场景；「随机散的东西」（装饰）留全局**。

## 7. 可选精炼

1. **共享候选池 `SlotPool`（.tres）**：17 种车 / 6 种树会在几十个槽里重复。做 `all_vehicles.tres` / `all_trees.tres`（`Array[PackedScene]`），槽位 `pool` 字段引用，避免每个车槽都拖一堆场景。特定集（5 种村屋）仍用 inline `choices`。
2. **野外用 `ScatterRegion` 节点**：森林程序散树，给森林模板放一个区域节点（count + 范围 + 候选池），`build()` 时区域内随机撒 N 棵，使野外/城镇接口统一。
3. **模板骨架一键生成**：用原版 `dx/dy` 数据写个工具脚本，一次性在模板场景里铺好 `TemplateSlot`，再手动微调——省去手摆几十个槽。
