### 1.4 感知半径（出处 `生物数值.md §5`）

- 多数僵尸**未显式 SetRange**，依赖 Turret/LOS 行为默认参数（◻️，可后续从 `objects/object_*.json` 的行为数组解出）。已确认显式值：
  - 坦克 LOS/Turret = **400**（行 40119/40126/40131）；冲锋检测 = 400。
  - 狼 Turret/LOS 默认；`perk_hunter=1` 时降到 **250**。
  - 鹿 LOS = **320**（hunter→250）；兔 LOS = **320**（hunter→250）；乌鸦惊飞距离 = **300**。
- 发现玩家：`距离 ≤ Player_Hear_radius`(=**1000**) 时播 spotted 音效，并 `noice(玩家X, 玩家Y, 400)` 把噪声广播给周围僵尸（行 39146-39148）。
- 尖叫者：看到玩家后每 0.5s `noice(玩家, 1700)`（行 40003）——核心威胁。

> 实现默认侦测半径：普通/军队/快速/掘地暂统一取一个工程值（建议 **400**，与坦克/spotted 广播半径对齐），写进各自 `.tres`，后续若解出原版默认再校准。

### 1.6 击杀奖励 / 掉落（出处 `生物数值.md §7`）

- 每杀 1 僵尸：`expierence_points += 8`（任意武器，行 37707，交叉引用 `09-经验与等级`）+ `Stats_Zeds_killed += 1`。
- 玩家武器命中僵尸：`HP -= 武器伤害(±25%)`；近战暴击/爆头 **×3**（常秒杀），暴击率 = 武器档(5/10/20%) + 武器 iv[9] + perk_header(+10%)。属 **06** 计算，本文只**接收** `take_damage`。
- 陷阱：捕兽夹对普通僵尸 **200**、坦克 200、狼 100（行 37147）。莫洛托夫燃烧每 0.5s 对僵尸 10。
- 掉落池（死亡时 `Spawn_drop(choose(...))`，物品 ID，出处 §7.1）：
  | 击杀对象 | 掉落池（物品 ID） |
  |---|---|
  | 普通 / 掘地 | 2,6,33,8,9,30,18,20,26,20,13,58,16,17（食物/饮料/绷带为主） |
  | 军队 | 3,11,12,14,15,69,21,20,43,34,58（偏弹药/配件） |
  | 快速 | 2,6,33,8,9,30,26,20,13,58,16,17 |
  | 射手 | 按 iv[8] 掉对应枪+弹（弹药 `round(random(10,30))`，1/3 概率附加掉落） |
  | 坦克 | 11,14,15,101 之一 + 固定生成尸体特殊物 |
  > 掉落实现依赖 `战利品/物品系统`（见 `02-环境物体场景 §3` 的 LootPoint，当前项目尚无统一刷新系统）——本文 P2 阶段接入，P0/P1 先只发 XP。

## 2. 与现有代码的关系

- **复用 `CharacterStatus`**（`status/character_status.gd`）：所有生物的 HP/最大 HP/`speed`/`died` 信号走它。僵尸死亡 = `health` 归 0 → `died` 信号 → 状态机切 Dead。
- **复用 `State`**（`state/state.gd`）：AI 状态机沿用玩家状态机同款写法（`character/player/state/player_state_machine.gd`：`states` 名单 + 各 State 节点 `finished.connect(set_state)` + `current_state.run()`）。
- **不照搬原版「控制器+皮肤」双实例**：原版 t247 控制器 / t941 皮肤拆成两个对象（控制器带 Bullet+LOS+Turret+Timer，皮肤负责显示/受伤）是 Construct2 行为系统所迫。本项目用**单个 CharacterBody2D** 内含状态机 + NavigationAgent2D + 两个 Area2D（Hitbox/Detection）即可，所有 iv 状态合并进脚本属性。
- **不照搬距离 create/erase**：与 `01-地图生成` 决策一致——一次性实例化 + `VisibleOnScreenEnabler2D` 暂停离屏 AI 的 `process`/`physics`。但**保留**原版重生半径规则（60s / 1500–2000px）作为**刷新器规则**（见 §6）。

### 2.1 战斗契约（必须与 `06-战斗与武器` 对齐）

- 僵尸必须实现**与玩家相同的受击方法**：`take_damage(amount: float, flags) -> void`，使 06 的玩家武器能命中它。
- 僵尸加入 `"zombies"` 组（group），玩家武器/陷阱用 `get_tree().get_nodes_in_group("zombies")` 或重叠检测命中。
- 僵尸攻击玩家：调用玩家的同一受击管线（传 `"zed_normal"/"zed_army"/"zed_fast"` 类型 + 伤害值），由 06 计算护甲减免与出血。

## 3. 资源设计：`CreatureResource`(.tres)

每个生物类型一份 `.tres`，把「数值 + 行为开关」从场景里抽出来——一个生物场景读不同 `.tres` 即变成不同类型（避免做 N 套近乎相同的派生场景）。

```gdscript
class_name CreatureResource extends Resource

enum Faction { ZOMBIE, ANIMAL }          # 阵营：决定加入哪个 group
enum AttackKind { MELEE, RANGED, NONE }  # 近战 / 远程(射手) / 无攻击(尖叫者、鹿兔)

@export var id: StringName                # "zed_normal" / "wolf" ...
@export var faction: Faction = Faction.ZOMBIE
@export var max_health: int = 60          # 出处 §1.1
@export var chase_speed: Vector2i = Vector2i(90, 100)   # random(min,max)
@export var wander_speed: Vector2i = Vector2i(80, 90)
@export var detection_radius: float = 400.0             # 视线/侦测，§1.4
@export var attack_kind: AttackKind = AttackKind.MELEE
@export var attack_interval: float = 1.0  # §3.3 近战每 1s
@export var damage_type: StringName = &"zed_normal"     # 传给 06 受击管线
# —— 行为标志（取代原版分散 iv）——
@export var flees_from_player := false    # 鹿/兔/乌鸦=true
@export var is_screamer := false          # 尖叫者：每 0.5s 噪声广播
@export var is_burrower := false          # 掘地：出场播 jump 埋伏
@export var can_charge := false           # 坦克冲锋
@export var noise_on_spot := 400.0        # 发现玩家时广播半径
@export var xp_reward := 8                 # §1.6
@export var loot_pool: PackedInt32Array   # §1.6 掉落池（P2 接入）
@export var sprite_frames: SpriteFrames   # 动画（idle/walk/attack_up..down/dead）
```

预置资源（`creature/resources/`）：`zed_normal.tres` / `zed_army.tres` / `zed_fast.tres` / `zed_shooter.tres` / `zed_buried.tres` / `zed_screamer.tres` / `zed_tank.tres` / `wolf.tres` / `deer.tres` / `rabbit.tres` / `crow.tres`。

> 射手需额外字段（武器型号 `iv[8]`、连发数、间隔、单发伤害）——可在 `CreatureResource` 加 `@export var gun: GunProfile`（远程数值，复用 06 的武器资源）或单独 `ShooterResource extends CreatureResource`。**已倾向**：射手用派生资源 `ShooterResource`，承载 §1.3 表。

## 4. 生物基类场景 + AI 状态机

### 4.1 节点结构

```
Creature (CharacterBody2D, script: creature.gd)        # group "zombies"/"animals"
├── Status            CharacterStatus                   # 复用现成；max_health/speed 来自 .tres
├── Nav              NavigationAgent2D                  # 寻路（§5）
├── Sprite           AnimatedSprite2D                   # frames 来自 .tres，按朝向切 4 向动画
├── Hitbox           Area2D                             # 受击体（被玩家武器命中），layer=敌人受击层
│   └── CollisionShape2D
├── Detection        Area2D                             # 视线/侦测圈，半径=detection_radius
│   └── CollisionShape2D (CircleShape2D)
├── AttackRange      Area2D（可选）                      # 近战重叠判定；或直接用 Hitbox 与玩家重叠
│   └── CollisionShape2D
├── VisEnabler       VisibleOnScreenEnabler2D           # 离屏暂停 AI（§7）
├── StateMachine     Node (script: creature_state_machine.gd)
│   ├── Idle         State
│   ├── Wander       State
│   ├── Chase        State
│   ├── Attack       State
│   ├── Flee         State   (动物专用)
│   └── Dead         State
└── Timers
    ├── WanderTimer  Timer    # choose(4,7,9)s 随机转向
    ├── AttackTimer  Timer    # 近战 1s / 射手按枪间隔
    └── NoiseTimer   Timer    # 尖叫者 0.5s 广播
```

- `Creature` 用 `CharacterBody2D` + `move_and_slide()`，速度向量由当前 State 写入（朝 `Nav.get_next_path_position()`）。
- 朝向：按 `velocity`（或到玩家角度）切 4 向动画（attack_up/down/left/right、walk_*），复现原版「按角度切 4 向」（§2）。
- `take_damage(amount, flags)`：扣 `Status.health`；命中即可由 06 算暴击/爆头 ×3。`Status.died` → 状态机切 `Dead`。

### 4.2 状态机（复用 `State` + 玩家状态机同款写法）

`creature_state_machine.gd` 仿 `player_state_machine.gd`：持 `states` 名单、各 State 的 `finished.connect(set_state)`、`run()` 调 `current_state.run()`。状态：

| 状态 | 进入条件 | 行为 | 退出 |
|---|---|---|---|
| **Idle** | 出生 / 丢失目标且无路径 | 站立播 idle，启 `WanderTimer` | 计时到 → `Wander`；侦测到玩家 → `Chase` |
| **Wander** | 巡逻 | 随机选附近点（NavigationAgent2D 导航），速度=wander_speed；原版「先降 20，3s 回 wander 速」用计时器复现 | 到点/计时到 → `Idle`；侦测到玩家 → `Chase` |
| **Chase** | Detection 检测到玩家（或被噪声招引） | `Nav.target_position = 玩家`，按 chase_speed 追；持续广播 spotted 噪声 | 进入攻击范围 → `Attack`；丢失目标超时 → `Idle` |
| **Attack** | 与玩家重叠/进近战圈（近战）或 `HasTarget`（射手） | 近战：每 `attack_interval`(1s) 调玩家受击；射手：按枪间隔连发（走 06 远程）；坦克：冲锋逻辑 | 玩家离开 → `Chase` |
| **Flee**（动物） | 侦测到玩家且 `flees_from_player` | 朝远离玩家方向冲刺 2s（鹿/兔/乌鸦） | 2s 后 → `Idle` |
| **Dead** | `Status.died` | 停物理、切尸体精灵、发 XP（→09）、掉落（→P2）、`queue_free` 或留尸 | 终态 |

特殊行为挂在对应 State：

- **掘地 Buried**：出生先进特殊「埋伏」子态——播 jump 出土动画后再转 `Chase`/`Wander`（`is_burrower`）。原版细节阈值未逐一展开（§12），先做「出土动画 → 转移动」。
- **尖叫者 Screamer**：`is_screamer` → 进 `Chase` 后启 `NoiseTimer`(0.5s) 调 `broadcast_noise(玩家位置, 1700)`（§5），自身无 `Attack`（`AttackKind.NONE`），速度低(30-40)且仅有视线时移动。
- **坦克 Tank**：`can_charge` → `Attack` 内含子状态机（普通追 110 / 冲锋 200 持续 2s 回 110），近战 `Zed_dmg×2`。**Boss 暂缓**（关卡 ≥1，见 §10）。
- **射手 Shooter**：`Attack` 用 06 的远程开火（`ShooterResource` 的枪型/间隔/连发），慢速接近（20-30）后开火。

## 5. 感知与噪声招引

### 5.1 视线（视觉）

- 用 **Detection (Area2D, CircleShape2D r=detection_radius)** 替代原版 LOS+Turret。玩家进圈 → `body_entered`（玩家在 `"player"` 组）→ 设目标、转 `Chase`。
- 进阶（可选）：用 `RayCast2D` 或 `NavigationServer2D` 可见性查询做真·视线遮挡（墙后看不到）。**P0 先用纯距离圈**，遮挡留 P1。

### 5.2 噪声招引（听觉）

原版用 `noice(x, y, radius)` 广播；本项目用**信号 + group ping**：

- 全局 `NoiseBus`（autoload 或 `WorldEvents` 信号）：`signal noise_emitted(world_pos: Vector2, radius: float)`。
- 噪声源：玩家开枪（06）、尖叫者（0.5s）、莫洛托夫点燃（半径 100），调 `NoiseBus.noise_emitted.emit(pos, radius)`。
- 僵尸订阅：收到信号后若 `自身到 pos 距离 ≤ radius` 且当前非 `Attack`，则把 `pos` 设为目标转 `Chase`（被招引）。
- 发现玩家时自身也广播（spotted，半径 400），把附近僵尸一起拉过来（复现 §1.4 行 39146）。

> 不做数值化 aggro/仇恨（原版亦无，§11）——目标就是玩家或噪声点，靠距离圈 + 噪声广播驱动。

## 6. 刷新系统（Spawner）

刷新点 marker 由 `01-地图生成`/`03-地块模板`在世界里放置（本文**不**负责放 marker，只消费）。每个 marker 带「类型/帧」字段决定刷什么（§1.9）。

```
ZedSpawnPoint (Marker2D, script: zed_spawn_point.gd)
- @export spawn_kind: int          # 对应原版刷新帧 0/1/3/4/...（§1.9）
- @export subtype: int             # 帧1 的军队子类等
- var alive: Array[Creature]       # 本点当前存活实例（取代原版 iv[0] 计数）
- RespawnTimer: Timer (60s)        # ZED_RESPAWN_TIME
```

刷新规则（复现 §1.8/§1.9，但用 Godot 计时器/距离判定）：

- **目标数 `Zed_per_base`**：从 `12-游戏模式`难度 + `Stats_time_lived` 升级表（§1.8）取当前值。
- **补刷条件**：`RespawnTimer` 每 60s（或某只死亡后）触发；仅当 `刷新点到玩家距离 > 1500 且 ≤ 2000(respawn_radius_npc)` 时 `Repeat(目标数 − alive.size())` 补足。
- **回收**：实体距玩家 `> 2000` 时销毁回收（伪持久），腾名额。**注意**：这条是「刷新器距离规则」，与「离屏性能门控」是两码事——门控只暂停 process，不销毁（见 §7）。原版超图边界(X<500 或 >17277)直接清除，本项目按世界边界做等价裁剪。
- **构成**：按 `spawn_kind` + 概率 `choose(...)` 选具体类型实例化（§1.9），实例化时给其 `CreatureResource`。
- **难度缩放**：伤害（§1.2）和数量（`Zed_per_base`）均读 `12-游戏模式`的当前难度；本文不持有难度状态。

### 6.1 性能门控（与 01 一致）

- 用 `VisibleOnScreenEnabler2D`（节点内 `VisEnabler`）在离屏时关掉该僵尸的 `process`/`physics`（`process_mode = PROCESS_MODE_DISABLED`），进屏恢复——**不销毁**。
- 渲染本身由引擎自动剔除。
- 仅在实测有瓶颈或 Web/移动导出时，才考虑按刷新点分块加载。

## 7. 寻路（NavigationServer2D + NavigationAgent2D）

- **推荐 NavigationServer2D + NavigationAgent2D**，不用自研 A*（项目 `test/astart_grid_test*` 仅为试验，网格 A* 难处理动态实体 + 自由像素坐标的世界）。
- 导航区域来源：由 `01-地图生成`产出的地面（草地/道路可走，建筑/障碍/水域为障碍）烘焙成 `NavigationRegion2D`/`NavigationPolygon`。建筑、障碍（`02-环境物体场景`的 StaticBody2D，layer 2）作为导航障碍（`NavigationObstacle2D` 或从 region 挖洞）。
- 因地图程序化生成且巨大（16320px），导航网格需**生成后烘焙**（`NavigationServer2D` 运行时 bake，或按 block 分块 region）。
- 僵尸：`Nav.target_position = 目标`，每物理帧取 `Nav.get_next_path_position()` 算 `velocity` → `move_and_slide()`。
- 动物逃跑可不依赖导航网格（直接朝反方向冲，撞墙由碰撞滑动解决），简化。

> 待定：导航网格是「全图一张大 region」还是「按 block 分块 + 运行时拼接」——见 §9。

## 8. 死亡 → 掉落 / 尸体

- `Status.died` → `Dead` 状态：
  1. 关物理/碰撞，切尸体精灵（原版按皮肤编号生成对应尸体）。
  2. **发 XP**：玩家 `expierence_points += xp_reward(8)`、`Stats_Zeds_killed += 1`（交叉引用 `09-经验与等级`）。
  3. **掉落**（P2）：按 `loot_pool` `choose` 生成物品，依赖战利品/物品系统（见 `02-环境物体场景 §3` LootPoint；当前项目无统一系统，P0/P1 跳过）。
  4. 尸体保留若干秒后 `queue_free`，或留作可搜刮尸体（视 02/loot 系统）。
- 射手掉落按 `iv[8]` 出对应枪+弹（§1.6）。坦克掉特殊物。

## 9. 决策

### 已定 ✅

- ✅ **架构**：单 `CharacterBody2D` 生物 + 内置状态机（复用 `State`）+ `CharacterStatus` + 两个 Area2D（Hitbox/Detection），**不**做原版「控制器+皮肤」双实例。
- ✅ **类型由 `.tres` 驱动**：一个生物场景读不同 `CreatureResource` 变不同类型（射手用派生 `ShooterResource`），不做 N 套近乎相同的派生场景。
- ✅ **复用现有基类**：HP/死亡走 `CharacterStatus`；状态机写法仿 `player_state_machine.gd`。
- ✅ **战斗契约对齐 06**：`take_damage(amount, flags)` + `"zombies"` 组；攻击玩家走 06 受击管线。
- ✅ **加载策略**：静态实例化（由刷新器产出）+ `VisibleOnScreenEnabler2D` 离屏暂停，**不**照搬距离 create/erase；但**保留** 60s / 1500–2000px 刷新器规则。
- ✅ **寻路**：用 NavigationServer2D + NavigationAgent2D，弃自研 A* 网格。
- ✅ **噪声**：用全局信号 `NoiseBus` + 距离判定广播，不做数值 aggro。
- ✅ **不做双精灵 skin**：原版皮肤切换并入单 `AnimatedSprite2D` 的不同动画/帧。

### 待定 ⏸️/❓

- [ ] **D-导航网格粒度**：全图一张大 `NavigationRegion2D` vs 按 block 分块运行时拼接？倾向**先全图 bake 一次**，性能不够再分块。
- [ ] **D-真·视线遮挡**：P0 用纯距离圈，是否补 RayCast/可见性查询做墙后失察？倾向 **P1 再补**。
- [ ] **D-Boss 范围**：坦克（HP 1000、冲锋状态机）做到哪一档？倾向 **暂缓**（关卡 ≥1 才出，先把 5 类普通僵尸 + 动物打通）。
- [ ] **D-掉落接入**：死亡掉落依赖尚不存在的战利品系统，P0/P1 仅发 XP，掉落随 02/loot 系统落地后接入。
- [ ] **D-默认侦测半径**：多数僵尸原版未显式设值（◻️），暂统一取 400，后续从 `objects/*.json` 行为数组解出再校准。
- [ ] **D-横竖朝向资源**：4 向动画用一套 `SpriteFrames`（按 velocity 切 animation）即可，无需像环境物那样拆横竖场景——确认资源帧是否齐 4 向。
