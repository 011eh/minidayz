## 3. 场景架构（按能力共享基础场景，沿用建筑的继承思路）

每个物体一个 `.tscn`（手绘碰撞最贴合），按**能力**复用几个小基础场景：

```
Obstacle (基)              StaticBody2D + Sprite2D ×2(SpriteH/V) + CollisionShape2D ×2(ShapeH/V)
  └─ 障碍 / 围栏 / hesco / pillar / 长椅 / 车壳 都继承

SearchableObject           = Obstacle + InteractionArea + LootPoint ×N
  └─ 车辆 / 垃圾桶 / lootbox

Tree                       躯干碰撞(待定) + Canopy(Sprite2D, 高 z_index 盖玩家)
HarvestPlant               AnimatedSprite2D(生长帧) + InteractionArea, 无碰撞
WaterPump                  Sprite2D + InteractionArea
```

- **变体**：色/朝向/损坏用同一 `.tscn` 的 spritesheet 帧；同类多资源（4 种 leaves、多色车）放进 `TemplateSlot.choices` 由槽位随机选（对应原版 `choose`）。
- **共享候选池**（可选）：17 种车 / 6 种树会在几十个槽重复，后续可做 `all_vehicles.tres` / `all_trees.tres`（`Array[PackedScene]`）减少重复拖拽。

### `LootPoint` 节点设计（贯穿车辆/容器）

```gdscript
@tool
class_name LootPoint extends Marker2D
@export var loot_type: int = 1   # 1–21，对应原版 iv[0] 物品池
# 编辑器画个小图标；运行时由全局刷新系统读取，本身不渲染
```

> 物体场景里只**摆锚点**；刷新逻辑（按 `loot_type` 选池 + 距离刷新 + 清理远处地面物）属于独立的「战利品刷新系统」，**当前项目尚无此系统**，按 §7 决策决定是否本轮一并做。

## 3.5 已实现：基础场景（基座）✅

> 位置：`environment/objects/`。这些是**基座**，具体障碍/车/树由其派生（New Inherited Scene，或 instantiate child + 可编辑子节点）。对应 §5 的 P0-0 / P0-1。

### 共用脚本 `obstacle.gd`（障碍 + 车辆 + 容器复用）

`@tool class_name Obstacle extends StaticBody2D`。横竖朝向各一套**贴图 + 碰撞**节点（`SpriteH`/`SpriteV` 与 `ShapeH`/`ShapeV`），`orientation` 决定显示哪套；`@tool` setter 让编辑器里改 `orientation` 即时预览（横/竖均可见，摆地图所见即所得）：

| 导出属性 | 类型 | 作用 |
|---|---|---|
| `orientation` | `enum {HORIZONTAL, VERTICAL}` | HORIZONTAL 显示 `SpriteH`/`ShapeH`，VERTICAL 显示 `SpriteV`/`ShapeV` |
| `random_frame` | `Vector2i` | 随机帧 `[min,max]`（车换色/损坏）；`Vector2i.ZERO`=不随机（仅运行时） |

- 子节点固定命名：`SpriteH`/`SpriteV`(Sprite2D)、`ShapeH`/`ShapeV`(CollisionShape2D)，`get_node_or_null` 容错。
- `_apply_orientation()` 编辑器/运行时通用：切 Sprite 的 `visible`、切 Shape 的 `disabled`（`set_deferred`）。**只改显隐,不改贴图,保存零风险**。
- `random_frame` 仅运行时（`Engine.is_editor_hint()` 早退），作用于当前朝向的 Sprite。
- **碰撞用 `CollisionShape2D.shape`，不用 `CollisionPolygon2D`**：~90% 障碍是轴对齐矩形（含 lavka 横竖两帧），用 `RectangleShape2D`；少数复杂帧（`obst_barricade-sheet0` 破损帧、`pillar-sheet0` 大柱帧）改用 `Convex/ConcavePolygonShape2D`。

### ① `obstacle.tscn` — 通用障碍基座
```
Obstacle  StaticBody2D  (script, collision_layer=2, collision_mask=0)
├── SpriteH Sprite2D          (centered=false；横向贴图)
├── SpriteV Sprite2D          (visible=false；竖向贴图)
├── ShapeH  CollisionShape2D  (横向碰撞)
└── ShapeV  CollisionShape2D  (disabled=true；竖向碰撞)
```
> 无 H/V 之分的物体派生时删掉 `SpriteV` + `ShapeV` 即可。

### ② `vehicle.tscn` — 车辆残骸基座（复用脚本）
```
Vehicle  StaticBody2D  (script, collision_layer=2, collision_mask=0)
├── SpriteH      Sprite2D          (centered=false；random_frame 出颜色)
├── SpriteV      Sprite2D          (visible=false)
├── ShapeH       CollisionShape2D
├── ShapeV       CollisionShape2D  (disabled=true)
└── LootAnchors  Node2D            (P0 空占位；P1 塞 LootPoint，每车≤6)
```

### ③ `tree.tscn` — 树基座（无脚本）
```
Tree  Node2D  (原点在树干基部)
├── Trunk   StaticBody2D  (collision_layer=2)
│   └── TrunkShape  CollisionShape2D  (CircleShape2D r=8，置于基部中心)
└── Canopy  Sprite2D  (z_index=1 恒盖玩家上方；centered=false)
```

### 派生流程与约定
1. 右键基座 → **New Inherited Scene**（或 instantiate child + 可编辑子节点）。
2. 设 `SpriteH.texture`（横向贴图）+ 给 `ShapeH` 画形状（脚底对齐节点原点以保证 y_sort）；有竖向 → 设 `SpriteV.texture` + 给 `ShapeV` 画形状；车用 `random_frame` 出颜色。
3. 摆放时设每个实例的 `orientation`（编辑器即时预览横/竖）。派生场景丢进对应 `TemplateSlot.choices`。
- 承载父节点（`LocationTemplate/Slots`）需 `y_sort_enabled`，配合各物体"原点在脚底"，前后遮挡才正确。
- 只有横向的物体：删掉 `SpriteV` + `ShapeV`，`orientation` 保持 HORIZONTAL。
- 朝向分在**两个 PNG**（`hesco_h`/`hesco_v`、`fence_h`/`fence_v`）的物体：直接做成两个独立场景，无需用 `orientation` 切换。
- 基座空 `CollisionShape2D` 在编辑器报"无形状"警告 = 预期，派生填形状后消除。
- `tree.tscn` 的 `Canopy.offset` / `TrunkShape.position` 留默认 (0,0)，按各树贴图尺寸在派生场景微调。
