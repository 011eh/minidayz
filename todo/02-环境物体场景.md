### 原版行为要点（带出处）

- **Loot 不内置在物体上**：物体旁放一个 `LootPoint`（原版 `t209`）/ 车用 `car_loot_point`（`t232`，每车 ≤6 锚点）；锚点 `iv[0]=1–21` 决定物品池，由全局 **1480s 距离刷新**统一刷。来源：`数值探索/建筑容器战利品表.md`、`objects/object_209.json`、`object_232.json`。
- **车壳**：实体碰撞；色/损坏是变体帧（intact/damaged），**不可破坏**。`objects/object_476~480.json`。
- **树**：原版 JSON 称树**无碰撞**，靠帧叠加让树冠盖在玩家上方（玩家可从树下穿过），无透明淡出，不可砍伐。`objects/object_174/200/201.json`。⚠️ 但资源单独有 `tree_block.png`（"block"=阻挡），与"无碰撞"矛盾——**见 §7 待定 D-树**。
- **障碍**：均有碰撞多边形；`obst_barricade` 有 broken 帧可破坏，其余仅地图清理（`DestroyOutsideLayout`），玩家不可破坏。`-sheet0/-sheet1` 是朝向/状态变体帧，**非游戏内损坏进度**。`objects/object_309/310/560.json`。
- **植物**：无碰撞、可采（莓果/甜椒/西葫芦/番茄）、多帧生长阶段、采后按计时重长。`objects/object_287/552/553/554.json`。
- **水泵**：可交互喝水/装水，单帧无碰撞；随机可能"没水"。`objects/object_435.json` + `l_eng_log.xml` ID12/15/16。

## 3. 场景架构（按能力共享基础场景，沿用建筑的继承思路）

每个物体一个 `.tscn`（手绘碰撞最贴合），按**能力**复用几个小基础场景：

```
obstacle.tscn (基)         StaticBody2D + Sprite2D(Sprite) + CollisionShape2D(Shape)，无脚本
  └─ 障碍 / 围栏 / hesco / pillar / 长椅 都继承

Vehicle (独立)             StaticBody2D + Sprite2D(网格帧) + Shape + LootAnchors
  └─ 车壳：颜色行 × 状态列帧网格，独立脚本 vehicle.gd

SearchableObject           = obstacle.tscn 派生 + InteractionArea + LootPoint ×N（待做）
  └─ 垃圾桶 / lootbox；车辆走 Vehicle 自己的 LootAnchors

Tree                       躯干碰撞(待定) + Canopy(Sprite2D, 高 z_index 盖玩家)
HarvestPlant               AnimatedSprite2D(生长帧) + InteractionArea, 无碰撞
WaterPump                  Sprite2D + InteractionArea
```

- **一物一朝向**：基座**不再内含横/竖两套节点**。有两个朝向的物体（block、barricade、sandblock、trash_container、lavka…）直接做成**两个独立派生场景**（`xxx_h.tscn` / `xxx_v.tscn`），与本就分两 PNG 的 `fence_h/v`、`hesco_h/v` 处理方式统一。
- **变体**：障碍的色/损坏在派生场景里直接选定 spritesheet 帧（`Sprite.frame`）；同类多资源（4 种 leaves、多色车、横竖两态）放进 `TemplateSlot.choices` 由槽位随机选（对应原版 `choose`）。
- **共享候选池**（可选）：17 种车 / 6 种树会在几十个槽重复，后续可做 `all_vehicles.tres` / `all_trees.tres`（`Array[PackedScene]`）减少重复拖拽。

### `LootPoint` 节点设计（贯穿车辆/容器）

```gdscript
@tool
class_name LootPoint extends Marker2D
@export var loot_type: int = 1   # 1–21，对应原版 iv[0] 物品池
# 编辑器画个小图标；运行时由全局刷新系统读取，本身不渲染
```

> 物体场景里只**摆锚点**；刷新逻辑（按 `loot_type` 选池 + 距离刷新 + 清理远处地面物）属于独立的「战利品刷新系统」，**当前项目尚无此系统**，按 §7 决策决定是否本轮一并做。

## 3.5 已实现：基础场景（基座）✅

> 位置：`environment/objects/`。这些是**基座**，具体障碍/车/树由其派生（New Inherited Scene，或 instantiate child + 可编辑子节点）。对应 §5 的 P0-0 / P0-1。

### 无脚本基座 `obstacle.tscn`（障碍 + 容器复用）

纯 `StaticBody2D` + **单套**贴图/碰撞节点（`Sprite` + `Shape`），**无脚本**。
> 原 `obstacle.gd`（`class_name Obstacle` + `random_frame`）已删除：基座逻辑已空转——`random_frame` 无任何派生场景使用，`_active_sprite()` 仅单处复用且 Vehicle 已不再继承，故退化为纯场景。车辆是独立的 `vehicle.gd extends StaticBody2D`（帧模型为二维网格）。**朝向不由脚本切换**——横/竖是两个独立派生场景（见下文）。

- 子节点固定命名：`Sprite`(Sprite2D)、`Shape`(CollisionShape2D)。
- **帧变体**：障碍的色/损坏帧在派生场景里直接设定 `Sprite.frame`（不再有运行时随机帧）；多资源/多朝向用 `TemplateSlot.choices` 随机选。
- **碰撞用 `CollisionShape2D.shape`，不用 `CollisionPolygon2D`**：~90% 障碍是轴对齐矩形，用 `RectangleShape2D`；少数复杂帧（`obst_barricade-sheet0` 破损帧、`pillar-sheet0` 大柱帧）改用 `Convex/ConcavePolygonShape2D`。

### ① `obstacle.tscn` — 通用障碍基座（无脚本）
```
Obstacle  StaticBody2D  (collision_layer=2, collision_mask=0)
├── Sprite  Sprite2D          (centered=false)
└── Shape   CollisionShape2D
```

### ② `vehicle.tscn` — 车辆残骸基座（独立脚本 `vehicle.gd`）
```
Vehicle  StaticBody2D  (script, collision_layer=2, collision_mask=0)
├── Sprite       Sprite2D          (centered=false；颜色行 × 状态列 帧网格)
├── Shape        CollisionShape2D
└── LootAnchors  Node2D            (P0 空占位；P1 塞 LootPoint，每车≤6)
```

### ③ `tree.tscn` — 树基座（无脚本）
```
Tree  Node2D  (原点在树干基部)
├── Trunk   StaticBody2D  (collision_layer=2)
│   └── TrunkShape  CollisionShape2D  (CircleShape2D r=8，置于基部中心)
└── Canopy  Sprite2D  (z_index=1 恒盖玩家上方；centered=false)
```

### 派生流程与约定
1. 右键基座 → **New Inherited Scene**（或 instantiate child + 可编辑子节点）。
2. 设 `Sprite.texture` + 给 `Shape` 画形状（脚底对齐节点原点以保证 y_sort）；障碍如需固定变体帧直接设 `Sprite.frame`，车用 `color_count`/`has_open_state` 出颜色与开关帧。
3. 有两个朝向的物体：做**两个派生场景** `xxx_h.tscn` / `xxx_v.tscn`，各填各自朝向的帧/形状。两个都丢进对应 `TemplateSlot.choices`。
- 承载父节点（`LocationTemplate/Slots`）需 `y_sort_enabled`，配合各物体"原点在脚底"，前后遮挡才正确。
- 单朝向物体只做一个场景即可（不带后缀或按需命名）。
- 朝向分在**两个 PNG**（`hesco_h`/`hesco_v`、`fence_h`/`fence_v`）或同一 PNG 不同 atlas 区域（lavka、sandblock、barricade）都一样：均做成两个独立场景。
- 基座空 `CollisionShape2D` 在编辑器报"无形状"警告 = 预期，派生填形状后消除。
- `tree.tscn` 的 `Canopy.offset` / `TrunkShape.position` 留默认 (0,0)，按各树贴图尺寸在派生场景微调。
