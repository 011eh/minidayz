## 1. 伤害契约（06 拥有）

**所有可受伤目标（玩家、僵尸、动物、可破坏部署物）共享同一受击方法**。这是 06↔07↔10↔11 的唯一耦合面。

```gdscript
# 伤害载荷（轻量 Resource 或 RefCounted；不用魔法码）
class_name DamageInfo
enum Source { BULLET, MELEE, ZOMBIE, TRAP, FIRE, COLLISION }
var amount: float          # 已结算的基础伤害（调用方先 roll ±25%）
var source: Source
var attacker: Node = null  # 伤害来源节点（用于仇恨/朝向）
var is_crit: bool = false

# 受击方：玩家状态 与 僵尸/动物/可破坏物 各自实现
func take_damage(info: DamageInfo) -> void
```

**规则**：
- 单参数永远是 `DamageInfo`，**不得**用 `take_damage(amount)` 或 `take_damage(amount, flags)`，**不得**叫 `apply_damage()`。
- 出血/护甲/格挡/难度减伤的结算在**接收方**的 `take_damage()` 内按 `source` 分支完成（公式见 `06` §6）。
- **命中机制 = 碰撞层**，不是组：**layer 3 = 玩家受击体**、**layer 4 = 敌人受击体**（世界实体仍 layer 2，交互 layer 1）。玩家武器命中 layer 4，僵尸攻击命中 layer 3，各自调对方 `take_damage()`。
- 组仅用于查询/广播（噪声聚怪、刷新计数）：僵尸 `"zombies"`、动物 `"animals"`、玩家 `"player"`。两者都实现 `take_damage`。

> 调用样例（僵尸打玩家）：`player_status.take_damage(DamageInfo.new(dmg, DamageInfo.Source.ZOMBIE, self))`。

## 2. 热源契约（08 拥有）

**篝火/帐篷/车厢**等暖源统一为 `HeatSource`，体温系统（08）用 `Area2D + 分组` 发现并叠加保暖。

```gdscript
# 暖源节点（挂在篝火/帐篷/载具车厢上）
class_name HeatSource extends Area2D   # 加入组 "heat_source"
@export var heat_value: float = 4.0    # 每 tick 体温加成（统一字段名）
@export var active: bool = true        # 开关（篝火点燃/熄灭、车有油未损毁）
@export var is_eave: bool = false      # 是否同时遮雨（屋檐/帐篷/车厢 → 抵消 rain_cold）
```

**规则**：
- 属性名统一 `heat_value`（**不是** `warmth`、**不是** `iv[0]`）。
- 体温系统通过玩家身上的 `HeatProbe`（Area2D）`get_overlapping_areas()` 筛 `is_in_group("heat_source")`，**累加**所有 `active==true` 热源的 `heat_value`（**原版逐个 `AddVar` 叠加，不取最强**：篝火 +15、暖区 +`t337.iv[0]`、地堡 +5、车厢 +4 同时生效可叠满；见 08 §5.4 与 decoded `game_events` 14803–14832）。
- 消费方**不得硬编码 +4**——载具车厢用 `heat_value=4` 字段表达，篝火/帐篷各自填字段。
- `is_eave==true` 的暖源额外抵消当前 `rain_cold`（交叉引用 08 体温公式）。

## 3. 进度天赋契约（09 拥有）

`Progression`（autoload）暴露**可被动拉取**的接口，其它系统**主动查询**，09 不反向耦合任何系统。

```gdscript
# autoload Progression
func add_xp(amount: int) -> void                                   # +XP 唯一入口（07 击杀调 add_xp(8)）
func has_perk(perk_id: StringName) -> bool                          # 是否已学某天赋
func perk_modifier(key: StringName, default: Variant) -> Variant    # 取天赋数值修正（无则返回 default）
func buy_perk(perk_id: StringName) -> bool                          # 花 XP 学天赋
signal xp_changed(xp: int)
signal perk_learned(perk_id: StringName)
signal level_up(level: int)
```

**消费方各自拉取**（不耦合）：
- `06`：`perk_modifier(&"hunter_range", 0)`、`has_perk(&"melee_master")`。
- `08`：`has_perk(&"snowborn")`（暖屋）、`has_perk(&"night_vision")`。
- `04`/`10`：`has_perk(&"agronom")`（免作物枯萎）、`perk_modifier(&"farmer_yield", 1.0)`。
- `inventory`：`perk_modifier(&"backpack_slots", 0)`（背包扩容）。

> 天赋 id 用 `StringName` 字面量，**不用**原版帧索引整数。完整 16 天赋 id 表见 `09` §3。

## 4. 难度配置契约（12 拥有）

`GameConfig`（autoload）持有难度与模式，**数值表是其它系统缩放的唯一来源**。

```gdscript
# autoload GameConfig
enum Difficulty { NOVICE = 0, REGULAR = 1, VETERAN = 2 }
enum GameMode { NORMAL, GUNNER_ZEDS, VIP, MORE_ZEDS, WILDLIFE, STALKER, PUBG, CASUAL, TARGET_DROP }
var difficulty: Difficulty
var game_mode: GameMode
func scale(key: StringName) -> float    # 取当前难度对应数值

# DIFFICULTY_TABLE（Novice / Regular / Veteran）—— 与原版一致
#   zed_dmg        6 / 9 / 10
#   zed_army_dmg   9 / 13 / 15
#   zed_fast_dmg   6 / 6 / 6   （固定，不随难度）
#   wolf_dmg       6 / 8 / 10
#   dmg_reducer    7 / 5 / 3   （玩家减伤因子）
#   night_cold     1 / 2 / 3
#   rain_cold      1 / 2 / 3
#   day_cold       6 / 7 / 8
#   starving_5s    0.75 / 1.1 / 1.4   （饥饿衰减 /5s；本项目 1s tick 折算 = 0.15 / 0.22 / 0.28 /s）
#   thirsty_5s     0.75 / 1.1 / 1.4   （水分衰减 /5s；折算同上）
#   regen_5s       2.0 / 1.75 / 1.5   （回血 /5s）
# 完整表（含上述衰减/回血及 /5s→/1s 折算约定）由 12 §2 拥有，本文仅摘要。
```

**规则**：`07`（僵尸伤害/刷新数）、`08`（寒冷速率）、`05`（PUBG loot 变体）一律读 `GameConfig.scale(key)`，**不得**在本系统内另存难度数值。审计确认 07/08 现用数值与本表一致。
> ⚠️ **饥渴衰减勘误**：状态系统的饥渴衰减/回血**同样**读本表——现有 `player_status.gd` 硬编码 `0.24/s`（不分难度，原版应为难度分档 `0.15/0.22/0.28 /s`），接入时须按 `GameConfig` 折算值替换（详见 12 §2/§3）。

## 5. 战利品契约（05 拥有）

```gdscript
# autoload LootSystem
func roll(pool_id: int) -> int                 # 按池号抽一个物品 id（重复条目即权重）
func spawn_at(pos: Vector2, pool_id: int)      # 在世界生成一个地面 Item
# LootPoint（Marker2D，挂容器/车/残骸内）：@export var loot_type: int  # 1..21
```

**池号语义**（关键区分，避免 07 误用）：
- **容器/车/残骸/地堡/空投** 走 `LootSystem` 的**池号**（`loot_type` 1..21；地堡箱 51/52；空投走 05 §7 独立分层表）。`14` 直升机残骸用池 **15**，地堡箱用 **51/52**——与 05 一致。
- **僵尸/动物死亡掉落**是**直接物品 id 列表**（`07` §1.6 的 `2,6,33,8,9,…` 是**物品 id 不是池号**），属独立机制，**不经** `LootSystem.roll`。两者并存，本文在此明确以免混淆。
- **钓鱼** `13` 用自己的 `FishCatchTable`，**不复用** loot 池。

## 6. 存档契约（12 拥有）

每个有持久状态的系统实现**自己那一半**：

```gdscript
func to_dict() -> Dictionary
func from_dict(d: Dictionary) -> void
```

`SaveSystem`（autoload）聚合：`PlayerStatus`（饥渴血温/出血/感染）、`Inventory`（物品存 `{id,count,durability}`，用 `ItemCreator` 重建）、`Progression`（xp/perks）、`World`（地图**种子**+时间+天气+已搜刮点+部署物+载具状态）、`Stats`（击杀/karma/解锁进度）。

**规则**：
- 地图**只存种子+关卡**，不存每个 tile；但**玩家改动过的世界对象**（部署物 `10`、已搜刮 loot 点 `05`、载具状态 `11`）必须显式进 `World`。
- 僵尸/动物是**瞬态**（读档后重刷），不存。
- 序列化方法名统一 `to_dict`/`from_dict`，**不得**叫 `save`/`load`。

## 7. 发物品契约（库存已实现）

给玩家发物品的**唯一标准链路**（库存系统已实现并接通）：

```gdscript
var it := ItemCreator.create_item_from_id(id)   # item/item_creator.gd
player.inventory.put_to_inventory(it, false)     # false = 不在世界树里
```

**前置代码缺陷（CRITICAL，实现前必修）**：`item/item_creator.gd:9 create_item_from_id()` 缺手枪段 `range(190, 202)`——**刷出/发放手枪(190–201)会 `assert` 崩溃**。补一行：
```gdscript
elif item_id in range(190, 202): return create_item(Pistol, item_id)
```
影响：`05`（枪械 loot）、`06`、`14`（空投/军械）。`13` 鱼 id 72–77 落在 0–85 段，不受影响。

> 另一桩：`inventory.find_item()`(inventory.gd:106) 返回 null，按 id 找物时（合成/换弹）需补——见 `06` §8。
