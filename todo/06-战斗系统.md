## 2. 「可受击」(Damageable) 契约 — 06↔07 接合点

定义一个**所有可受伤目标共享**的方法签名，玩家状态与僵尸各自实现。这是本文与 `07-僵尸AI.md` 的唯一耦合面。

### 2.1 伤害标志（DamageInfo）

不用魔法码，用一个轻量 `Resource` 或字典携带伤害元数据，决定出血概率/暴击/穿甲分支：

```gdscript
# combat/damage_info.gd
class_name DamageInfo extends RefCounted

enum Source { BULLET, MELEE, ZOMBIE, TRAP, FIRE }  # 决定出血概率分档（§6.2）

var amount: float          # 已 roll 完 ±25% 的最终预算伤害（进护甲前）
var source: Source
var attacker: Node         # 射手/挥击者（用于经验归属、朝向）
var is_crit: bool = false  # 暴击（命中时已判定，×3 见 §6.3）
```

### 2.2 受击接口（统一签名）

```gdscript
# 任何可受击目标都实现这一条：
func take_damage(info: DamageInfo) -> void
```

| 实现者 | 位置 | 行为 |
|---|---|---|
| **玩家** | `PlayerStatus`（或 `Player` 转发给 status）| 走玩家减伤公式（护甲 + 难度 mitigation）→ 扣 `health` → roll 出血 → 扣衣物耐久（§6.1 / §7.2）|
| **僵尸** | 07 的 `Zombie extends CharacterBody2D`（`zombies` 组）| 扣自身 HP → roll 出血动画（可选）→ 死亡发尸体/掉落（07 负责）|

- **组**：所有敌人加入 `zombies` 组；命中检测可用 `is_in_group('zombies')` 或直接鸭子类型 `has_method('take_damage')`。
- **建议**：玩家侧也提供 `Player.take_damage(info)` 作为僵尸调用入口，内部转发到 `PlayerStatus`，让"攻击者只认 `take_damage`"成立（双向对称）。

> 原版对照：玩家走 `Player_get_hit(type, dmg)`（生物数值 §6），僵尸走 `t941.iv[0] -= 武器伤害±25%`（生物数值 §7）。本文用统一 `take_damage(DamageInfo)` 替代两套魔法码入口。

## 3. Godot 节点结构

### 3.1 远程武器开火（挂在 Player 上）

枪口/弹道挂玩家而非武器节点（武器节点是背包 item，不在场景树渲染位置）。

```
Player (CharacterBody2D)
├── ...（已有 AnimationTree / Sprites / Inventory / StateMachine / DetectionArea）
├── Muzzle (Marker2D)              枪口位置锚点，随 last_direction 旋转/偏移
│   └── ShotRay (RayCast2D)        命中检测射线（hitscan），enabled=false，开火时单帧启用
├── MuzzleFlash (节点/GPUParticles) 枪口闪光（可选观感）
└── FireSound (AudioStreamPlayer2D)
```

- 开火/弹道由**状态机（aim_state）驱动**，调用 `RangedWeapon` 上的开火方法或一个 `RangedFirer` 组件（见 §4 决策）。

### 3.2 近战命中体（挂在 Player 上）

```
Player
└── MeleeHitbox (Area2D)           collision_layer=0, collision_mask=敌人受击层
    └── CollisionShape2D           扇形/矩形，monitoring=false（默认关）
```

- 命中体在挥击动画的"命中帧"启用一小段（AnimationPlayer 调用方法轨 或 定时窗口），见 §5。

### 3.3 敌人受击体（07 实现，本文约定层）

```
Zombie (CharacterBody2D, group=zombies)
└── Hurtbox (Area2D)               collision_layer=敌人受击层；被 MeleeHitbox/ShotRay 命中
```

### 3.4 碰撞层规划（新增）

| layer | 名称 | 占用者 | 被谁检测 |
|:--:|---|---|---|
| 1 | 交互检测（已有）| InteractionArea | — |
| 2 | 玩家交互/世界实体（已有）| 建筑/障碍 StaticBody、玩家本体 | 玩家移动 mask、ShotRay（子弹撞墙）|

## 4. 远程射击（hitscan + 动态散布）

### 4.1 命中方式：hitscan（RayCast2D） vs 投射物场景

**决策：用 hitscan（射线命中），不做飞行子弹场景。** 理由：

- 原版是**即时命中**子弹（生物数值 §4 射手 `Player_get_hit("bullet", …)` 在开火帧即结算；物品数值 §6.0-C 弹道角 = 瞄准角 ± random(散布)），俯视街机手感本就"指哪打哪一瞬到"。
- hitscan 用 `RayCast2D`（每开一枪设方向 → `force_raycast_update()` → 取碰撞体）最简、零生命周期管理。
- 例外：**弓/弩**（improvised_bow / crossbow）原版箭矢可拾回、慢速可见——这两把可后续单独做投射物场景（P1，见 §9）。主线先全 hitscan。

### 4.2 散布动态模型（复刻原版 5 参数，物品数值 §6.0-C）

`RangedWeapon.spread`（已声明的成员）作为**当前散布角**，在 `[basic_spread, max_spread]` 间动态变化。原版每 0.1s 更新一次（`Aiming` 组）：

| 触发 | 公式 | 钳制 | 原版出处 |
|---|---|---|---|
| 持续瞄准静止 | `spread -= cooldown_step` | 下限 `basic_spread`（≈`min_spread`）| §6.0-C "静止每 tick −cooldown" |
| 移动中 | `spread += run_penalty` | 上限 `max_spread` | §6.0-C "移动每 tick +run_penalty" |
| 每开一枪 | `spread += spread_per_shot` | 上限 `max_spread` | §6.0-C "每枪 +spread_per_shot" |
| 实际弹道角 | `aim_angle ± deg_to_rad(randf_range(-spread, spread))` | — | §6.0-C |

> ⚠️ 字段名映射：原版 `cooldown_step`（冷却步长）→ 本项目资源**未单列**该字段。两种处置（§10 待定 D-cooldown）：(a) 复用一个固定回复速率常量；(b) 后续给 `RangedWeaponResource` 补 `cooldown_step`。原版各枪 cooldown 见 `parsed/weapon_dispersion.json`（手枪 2–4 / 步枪 1–2，节选见物品数值 §6.1）。
> 准星 UI 按 `spread / max_spread` 分 5 档张合（物品数值 §6.0-C）——属 HUD，可与 `09-UI` 联动，本文先不做。

### 4.3 开火流程（接进 aim_state）

`aim_state.run()` 当前只在 `Input.is_action_pressed('aim')` 时瞄准。**在其内增加：持瞄 + 按 `attack` → 尝试开火**：

```
持有 aim 时：
  瞄准（已有）
  若 Input.is_action_pressed('attack') 且 fire_cooldown 到期：
    weapon = 当前装备的 RangedWeapon（按 weapon_state 从 inventory 取）
    若 weapon.get_round_count() <= 0: 播空仓音/触发换弹，return
    fire()
```

`fire()`（建议放 `RangedWeapon.fire(origin, aim_dir) -> DamageInfo` 或独立 `RangedFirer`）：

1. **射速门控**：用 `attack_rate` 起一个 cooldown（`Timer` 或累加 delta）。全自动持 `attack` 连发；半自动/栓动需松开再按（原版 `iv[0]` 就绪标志）。
2. **扣弹**：`round.number -= 1`（霰弹仍 −1，但生成多发弹丸，见 4.4）。
3. **散布扣加**：`spread += spread_per_shot`（钳 `max_spread`）。
4. **弹道**：`final_angle = aim_dir.angle() + deg_to_rad(randf_range(-spread, spread))`；设 `ShotRay.target_position` = 方向 × 射程 → `force_raycast_update()`。
5. **命中**：`ShotRay.get_collider()` 若在 `zombies` 组 / 有 `take_damage` → 进 §6 伤害结算 → `target.take_damage(info)`。
6. **磨损**：`durability -= resource.wear_per_use`（§7.1）。
7. **观感**：`MuzzleFlash` 一闪、`FireSound` 播放、画一条短暂 tracer（Line2D，可选）。

### 4.4 霰弹 / 弓弩 / 全自动 三类节奏（物品数值 §6.0-D）

| 类型 | 武器例 | 节奏 | 弹丸 |
|---|---|---|---|
| 全自动 | M4/AKM/MP5K/RPK… | 持 `attack` 每 `attack_rate` 连发（≈460–850 RPM）| 单发 |
| 半自动/栓动 | 手枪/SKS/Mosin/SVD | 按一次开一发，松开复位 | 单发 |
| 霰弹 | IZH-43 / Remington 870 / Saiga-12 | 单次扳机 | **多发弹丸**，各自带 random 散布（多条 ray 或多次 raycast）|
| 弓/弩 | improvised_bow / crossbow | 单发慢速 | 投射物（P1 例外，见 4.1）|

### 4.5 伤害数值（物品数值 §6.2，复用资源 min/max）

伤害 = `randi_range(min_damage, max_damage)`（资源里 min/max 即"基础±25%"的范围，原版 `round(基础 ± 基础/4)`，物品数值 §6.0-B）。关键样本（完整 ~40 把见物品数值 §6.2 / §6.3，**勿逐行复制**）：

| 武器 | min–max | 弹匣 | 备注 |
|---|:--:|:--:|---|
| PM | 18–30 | 8 | 手枪基准 |
| Magnum | 33–57 | 6 | 高伤手枪 |
| AKM | 26–44 | 30 | 450 RPM |
| Mosin | 67–113 | 5 | 栓动狙击 |
| SVD | 60–100 | 10 | 半自动狙 |
| Remington 870 | 15–25 ×弹丸 | 6 | 霰弹×多发 |

> ✅ 这些 min/max 已写进各 `.tres`，本文直接 `randi_range(resource.min_damage, resource.max_damage)`。

### 4.6 爆头/精准（原版机制，可选）

## 5. 近战（Area2D 命中体 + 暴击 + 格挡）

### 5.1 命中体扫掠

`MeleeHitbox`(Area2D) 在挥击动画的命中窗内启用：

- **推荐**：MeleeAttack 动画里加一条 **AnimationPlayer 调用方法轨**，在命中帧调 `enable_hitbox()` / 帧末 `disable_hitbox()`。
- **退路**：状态机已有 `attack_timer`（`MeleeAttack.end` 接其 timeout），可在 `melee_attack_state.start()` 开命中体、短延时关。
- 命中体形状朝 `last_direction`（玩家朝向）放置（扇形近似用矩形/胶囊）。

### 5.2 攻速门控（复用现成 attack_timer）

状态机 `set_state('MeleeAttack')` 已用 `attack_timer.is_stopped()` 防连击：

```gdscript
# player_state_machine.gd 现状
if state_name == 'MeleeAttack':
    if attack_timer.is_stopped(): attack_timer.start()
    else: return
```

- **改进**：`attack_timer.wait_time` 应取自当前近战武器资源的 `attack_rate`（攻速；原版 §6.3 攻速列：刀 0.8 / 撬棍 1.2 / 大锤 1.8…，值越大越慢）。在 `start()` 时 `attack_timer.wait_time = weapon.resource.attack_rate`。

### 5.3 命中结算（一次挥击命中一个/多个目标）

`MeleeHitbox.body_entered`/`area_entered` 命中 `zombies` 组目标时：

1. 伤害 = `randi_range(min_damage, max_damage)`（资源 min/max，原版 §6.3）。
2. **暴击**：`randf() < crit_chance` → `is_crit=true`，伤害 ×3（原版 §7 近战暴击 ×3）。`crit_chance` 已在 `melee_weapon_resource.gd`。
3. 构造 `DamageInfo{source=MELEE, is_crit, attacker=player}` → `target.take_damage(info)`。
4. **磨损**：`durability -= resource.wear_per_use`（§7.1；刀 0.4 / 斧 0.2 / 大锤 0.1…，已在 `.tres`）。
5. 去重：一次挥击对同一目标只结算一次（命中集合 / `monitoring` 单帧）。

### 5.4 近战数值（物品数值 §6.3，复用资源字段）

`crit_chance`、`block_chance` 已是 `melee_weapon_resource.gd` 字段。关键样本（完整 13 把见物品数值 §6.3）：

| 武器 | min–max | 暴击 | 格挡 | 攻速 | 磨损/次 |
|---|:--:|:--:|:--:|:--:|:--:|
| Hunter Knife | 16–22 | 0.10 | 0.05 | 0.8 | 0.4 |
| Crowbar | 22–28 | 0.25 | 0.15 | 1.2 | 0.3 |

## 6. 伤害结算管线（统一公式）

一条管线，攻击方与受击方各管一段。**roll 在攻击方，减伤在受击方。**

```
攻击方（fire / melee 命中）
  伤害 = randi_range(min_damage, max_damage)        # 基础±25%（已存 .tres）
  若暴击/爆头: 伤害 ×= 3
  → DamageInfo{amount=伤害, source, attacker, is_crit}
  → target.take_damage(info)

受击方 take_damage(info):
  ├─ 玩家:  扣血 = mitigate_player(info)             # 见 6.1
  └─ 僵尸:  扣血 = info.amount                        # 直接扣 HP（07）
  health -= 扣血
  若 health > 0:  roll 出血(info.source) → 若中则置 is_bleeding（见 6.2）
  受击连带: 玩家扣随机衣物耐久（见 7.2）
```

### 6.1 玩家减伤（原版 `Player_get_hit`，生物数值 §6.1）

```
基础减伤:  扣血 = info.amount(±随机 [-1,0,1]) − 护甲(armor)
近战完全格挡（仅 source=MELEE/ZOMBIE 近战）:
  按穿戴衣物类型查 block%（见下表），若 block% ≥ random(100) 则完全免伤(扣血=0)
格挡失败时穿甲下限:  扣血 = max(5, info.amount − round(armor/2))   # 原版 line 48967
```

**衣物格挡率（按衣物类型，原版 §6.1 / 物品数值 §5）：**

| 衣物档 | block% | 物品数值对应 |
|---|:--:|---|
| 轻型 | 5% | T恤/衬衫类（type 7,8,9）|
| 中型 | 15% | 夹克/卫衣类（type 3,4,6）|
| 战术 | 25% | 战术背心/部分护甲（type 1,5,15）|
| 重型 | 35% | 防弹衣/凯夫拉（type 2,10,11,12,16）|

> 护甲 `armor` = 各衣物 flat 减伤（物品数值 §5：军用头盔 3 / 防弹衣 3…，多件叠加）。

**难度减伤因子（`Difficulty_damage_reducer`，生物数值 §3.1）：**

| 难度 | 因子 |
|---|:--:|
| Novice 新手 | 7 |
| Regular 普通 | 5 |
| Veteran 老兵 | 3 |

> ⚠️ 原版该因子用于**僵尸伤害缩放/玩家减伤**（生物数值 §3.1）。本项目难度系统未做 → 先取一个默认难度常量（建议 Regular=5），实装难度后再读全局。具体作用点（除护甲外的额外减除项）按原版 `Player_get_hit` 公式接入。

### 6.2 出血判定（原版 §6.2，命中且未死才 roll）

出血一旦触发：`PlayerStatus.is_bleeding = true`（玩家侧；通路已通，每秒 `bleed_rate=0.82` 扣血）。原版出血计时 60s，靠绷带/破布止血（物品数值 §4：ID18/26）。

| 来源 `DamageInfo.source` | 出血概率 | 原版判定 |
|---|:--:|---|
| BULLET（子弹击中）| **50%** | `random(3) ≥ 1.5` |
| MELEE（人类近战武器击中玩家）| **25%** | `random(2) ≥ 1.5` |
| ZOMBIE（僵尸/狼近战击中玩家）| **≈16.7%** | `random(3) ≥ 2.5` |

> Godot 实现：`randf() < 0.5 / 0.25 / 0.167`。僵尸侧是否表现出血（仅动画）由 07 决定；本表的"置 is_bleeding"是**玩家受击**分支。

### 6.3 暴击（攻击方，原版 §7 ×3）

- 近战：`randf() < crit_chance` → ×3。
- 枪械爆头：见 §4.6（简化为按暴击率 ×3，或先略）。
- 陷阱/特殊（捕兽夹对僵尸 200 等，生物数值 §7）属 07/陷阱系统，本文不展开。

## 7. 耐久磨损

### 7.1 武器磨损（每次使用，复用 `wear_per_use`）

每开一枪 / 每挥击一次：`durability -= resource.wear_per_use`（钳 `≥0`）。原版各枪 0.3–1.0、近战 0.1–0.4（物品数值 §6.0-A `iv[6]` / §6.3 磨损列），值已写进各 `.tres`（如 `mosin.tres` 0.8、`hatchet.tres` 0.2、`m4_carbine.tres` 0.4）。

- 耐久归 0 的后果（卡壳/不可用/伤害衰减）原版细节未明 → 先**仅扣值**，归 0 行为留待 §10 待定。

### 7.2 衣物受击磨损（玩家被打时，原版 §6.4）

玩家 `take_damage` 时，**随机选一件穿戴衣物**扣耐久：

| 情形 | 扣耐久 | 原版 |
|---|:--:|---|
| 普通/僵尸命中 | `round(randf_range(10,15))` | line 48717 |
| 狼命中 | `round(randf_range(10,25))` | §6.4 |
| 无护甲格挡时 | `round(randf_range(5,15))` | §6.4 |

> 衣物耐久下降 → 有效保暖随之降低（原版保暖 = `衣物满保暖 × 耐久/100`）。**保暖联动交叉见 `08-状态衰减与体温.md`**（若已建），本文只负责"扣衣物耐久"。

## 10. 决策

### 已定
- [x] **D-命中方式**：枪械用 **hitscan（RayCast2D）**，复刻原版即时命中；弓/弩例外可后做投射物（§4.1）。
- [x] **D-受击契约**：统一 `take_damage(DamageInfo)`，玩家与僵尸共用，replace 原版双套魔法码入口（§2）。roll 在攻击方，减伤在受击方。
- [x] **D-不新增资源字段**：消费已有 `min_damage/max_damage`（=基础±25%）、散布 5 参数、`crit_chance/block_chance`、`wear_per_use`，不再存"基础伤害"（§1.1）。
- [x] **D-开火驱动点**：开火逻辑接进 `aim_state`（持 `aim` + `attack`），与原版"瞄准态才能开火"一致；近战接进 `melee_attack_state` + 现成 `attack_timer`（§4.3 / §5.2）。
- [x] **D-磨损复用**：武器每次用扣 `wear_per_use`；玩家受击随机扣一件衣物耐久 `round(random(10,15))`（§7）。

### 待定
- [ ] **D-cooldown_step**：散布恢复速率资源里没有专门字段——(a) 用固定常量 / (b) 给 `RangedWeaponResource` 补 `cooldown_step`（原版每枪有独立值，`weapon_dispersion.json`）。倾向 **(b) 补字段**以保真。
- [ ] **D-子弹可视化**：纯逻辑命中 / 画一条 tracer（Line2D 短暂）/ 命中火花。倾向 **P1 加短 tracer + 命中特效**，P0 先纯逻辑。
- [ ] **D-难度因子来源**：难度系统未做 → 先取 Regular(=5) 常量，后接全局难度（§6.1）。
- [ ] **D-爆头**：俯视无头部区域 → (a) 按暴击率 ×3 / (b) 先略。倾向 **P2 再做**。
- [ ] **D-耐久归0后果**：先仅扣值，归 0 行为（卡壳/伤害衰减/不可用）P2 定。
- [ ] **D-受击体归属（玩家）**：玩家受击用 Area2D Hurtbox 还是直接 `Player.take_damage` 被僵尸调用？倾向 **直接方法调用**（僵尸近战是 1s 重叠结算，不需精细 Hurtbox；射手子弹用 ray/Area 命中 Player 受击层）。
