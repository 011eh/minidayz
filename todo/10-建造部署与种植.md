## 1. 与现有代码的关系（接线点）

- **合成已就绪**：`item/item_action_table.gd` 有 9 条硬编码 `CraftingRecipe` + 拖拽合成流。部署是合成**之后**的独立步骤——制作出"部署套件"物品后，在物品的右键动作菜单里出现一个 **`部署`** 动作（与现有 `Consume`/`Split`/`Eject`/`Load into main` 并列，均由 `create_options(item)` 产出 `ItemAction`）。
- **套件物品已存在**：`campfire_kit.tres`、`bear_trap.tres`、`claymore.tres`、`landmine.tres`、`barbed_wire.tres` 等 `NumberItem` 资源已就位。本文只需把每个套件 → 映射到一个"落地 .tscn"。
- **物品发放链路已通**（见 `00` §2 纠正）：`player.inventory.put_to_inventory(item, false)` 可用；落地确认时反向**消耗**套件物品（`item.number -= 1` 或 `item.queue_free()`），与合成里 `c1.consumed` 写法一致。
- **交互复用**：进入预览不是 `InteractionArea` 按键交互，而是一个**临时 UI/控制态**（`DeployController`），更贴近原版"拖动幽灵选址"。但落地后的物体若可搜刮/可交互，仍复用现成 `mechanics/interaction_area.gd`。
- **碰撞层约定**（见 `02` §4）：世界实体在 **physics layer 2**。围栏落地后是 layer 2 障碍；合法性检测对 layer 2 做重叠查询。

> 不照搬原版 `iv[21]`（锁定标志）、`t566`（幽灵）、`Building_mode` 整数状态机；用一个控制器节点 + 状态枚举 + 信号表达。

## 3. Godot 节点结构

### 3.1 部署控制器 `DeployController`

挂在玩家下（或作为玩家持有的子节点），管理一次部署会话的预览态。

```
Player (CharacterBody2D)
└── DeployController (Node2D)              deploy_controller.gd, class_name DeployController
    ├── Ghost (Sprite2D)                   预览幽灵，跟随光标；modulate 着色绿/红 + alpha
    │   └── OverlapArea (Area2D)           collision_mask = 2（世界实体层）
    │       └── CollisionShape2D           与落地物碰撞形状一致，用于重叠合法性
    └── BuildTimer (Timer)                 one_shot；耗时按部署物配置（3/5/0.5s）
```

- **状态枚举**：`enum State { IDLE, PREVIEW, BUILDING }`。
- **进入预览**：物品菜单点"部署" → `start_deploy(item)` → 读该套件的部署配置（落地 .tscn + 耗时 + 幽灵帧）→ 显示 `Ghost`，态转 `PREVIEW`。
- **每帧/每 0.1s 更新**：`Ghost.global_position = get_global_mouse_position()`（或玩家右侧 +50 初始，原版 `t566` 在玩家右侧），调用 `_check_legality()` 刷新 `Ghost.modulate`（绿 `Color(0,1,0,1)` / 红 `Color(1,0,0,0.5)`）。
- **确认**（左键/确认键，仅合法时）：态转 `BUILDING`，玩家锁定（不照搬 `iv[21]`，用一个 `player.is_busy` 或冻结输入），`BuildTimer.start(build_time)`。
- **`BuildTimer.timeout`**：实例化落地 .tscn 到世界、消耗套件物品、`update_inventory_ui`、`Ghost` 隐藏、态回 `IDLE`，发 `deployed(scene)` 信号。
- **取消**（右键/Esc）：隐藏 `Ghost`，态回 `IDLE`，不消耗物品。

```gdscript
# 合法性：距离 + 重叠 + 地堡（伪代码骨架，非最终）
func _check_legality() -> bool:
    if Ghost.global_position.distance_to(player.global_position) <= 100.0:
        return false
    if OverlapArea.get_overlapping_bodies().any(func(b): return b.collision_layer & 2):
        return false
    if _is_in_bunker(Ghost.global_position):
        return false
    if _is_tent and _overlaps_existing_heat_source():
        return false
    return true
```

### 3.3 `HeatSource` 节点契约（与 08 共享）

篝火、帐篷落地后必须向体温系统登记为热源。定义一个轻量节点，**08 的体温计算读它**：

```gdscript
@tool
class_name HeatSource extends Area2D
## 暖区。玩家进入范围时，08 体温系统按 heat_value 给予回温。
@export var heat_value: float = 4.0  ## 原版帐篷 t337.iv[0]=4；篝火另定（字段名以 00a §2 为准）
@export var active: bool = true      ## 篝火可熄灭→关闭供暖
@export var is_eave: bool = false    ## 是否遮雨（屋檐/帐篷→抵消 rain_cold）
# collision_layer 设独立"热源层"，玩家身上一个探测 Area2D 查询之；
# 或 08 直接遍历场景内 HeatSource，按 active && 玩家在范围内取最大 heat_value。
```

- **契约**：08 体温公式在"室内/取暖"项里，取玩家范围内 `active` 的 `HeatSource.heat_value` 之和或最大值（具体由 08 定）。
- 帐篷 `heat_value=4`（原版明文）；篝火数值待 08 核定。
- 篝火可有燃料/熄灭逻辑 → `active=false` 时停止供暖（交叉引用 08）。

### 3.4 陷阱基 `Trap`

```
Trap (Node2D)                          trap.gd, class_name Trap
├── Sprite (Sprite2D)                  布防/已触发两帧
├── TriggerArea (Area2D)               collision_mask = 敌方层（07 僵尸/动物）
│   └── CollisionShape2D
└── [landmine/claymore] ExplosionArea  触发时启用，做 AoE 伤害
```

- **布防**：落地即 armed（可加极短延迟防自伤）。
- **触发**：`TriggerArea.body_entered` → 检测进入者属敌方 → **经 `06` 伤害管线**对其结算伤害（bear_trap 单体夹击 + 定身；landmine/claymore 爆炸 AoE，交叉引用 `06`/`07`）。
- **一次性**：触发后切"已触发"帧、禁用 `TriggerArea`、`one_shot` 语义；bear_trap 可考虑可回收复用（待定）。
- **伤害归属**：不在本文定义伤害数值，对目标调用统一受击方法 `target.take_damage(DamageInfo.new(amount, DamageInfo.Source.TRAP, self))`（单参恒为 `DamageInfo`，见 `00a` §1）。

### 3.5 播种（本文拥有；生长交叉引用 04）

播种是**种子物品的使用动作**，不是新建控制器：

- 在 `item_action_table.gd` 的 `create_options(item)` 里，对种子 ID（47/48/68）追加 `ItemAction.new('种植', _plant)`。
- `_plant`：检查 §2.2 四条件（不在车内 / `tile==6` / 群系 ∈{2,10,11} / 不重叠暖区·不在地堡）→ 全满足则播音 + 玩家锁定 + 5s `Timer` → `timeout` 时 `instantiate()` 对应作物 .tscn（来自 04）到玩家脚下 → 种子 `number -= 20`。
- **农田 tile 检测**：取玩家 `global_position` → 地形 `TileMapLayer`（交叉引用 `01` ground 层，原版 `t945`）→ `local_to_map()` → `get_cell_*` 判定 tile 索引是否 = **6**。
- **作物实例**：复用 04 的 `crop.gd` / `crop_tomato|pepper|squash.tscn`，从生长阶段 0 起跑；**生长 tick、成熟、枯萎、农艺免疫全部是 04 的逻辑，本文不实现**。
