# 可驾驶载具设计

> 定位：本文是**可驾驶载具**（UAZ / Volga）——能进车、加油、驾驶、撞坏、爆炸、重生的活动实体。
> 调研来源：`origin/others/文档/数值探索/载具数值.md`（原版 `t946`，组 `Drivable vehicles`，`game_events.decoded.txt:22929-23681`）。
> 复用约定：进出车复用 `mechanics/interaction_area.gd`；驾驶手感镜像 `character/player/player.gd`；车厢供暖走 `08-体温昼夜天气` 的 `HeatSource` 契约；加油消耗 `item/res/number_item/gasoline.tres`。

---

## ⚠️ 与既有「车辆残骸」彻底区分（不要混淆）

| | **静态车壳残骸**（已存在） | **可驾驶载具**（本文） |
|---|---|---|
| 类 / 文件 | `Vehicle extends StaticBody2D`，`environment/objects/vehicle.gd` | 新建 `DrivableVehicle extends CharacterBody2D` |
| 原版对象 | car_regular/van/police… `object_476~480` | **`t946`**（UAZ / Volga） |
| 性质 | **布景 + 战利品**（不动、可搜刮、`open()` 开后备箱帧） | **活动载具**（可进车驾驶、有油/血/重生） |
| 归属文档 | `02-环境物体场景` / `05`（loot） | **本文 11** |
| 进模板 | `TemplateSlot.VEHICLE` 槽 | 地图/模板生成专门落子（见 §8，cross-ref `01`/`03`） |

> 二者**类名都叫 Vehicle 会冲突**——本文一律用 **`DrivableVehicle`**（`class_name DrivableVehicle`）。静态残骸保持现状不动。
> 一句话：静态残骸是"撞不动的破车 + 翻箱子"；可驾驶载具是"能开走的车"。两条线互不继承。

## 1. 数值 / 规格（已核对原版，带出处）

### 1.1 车型驾驶参数（`载具数值.md §2`，`23420-23480`）

| 车型 (`iv[0]`) | 最大速度 MaxSpeed | 加速 Accel | 减速 Decel | 转向 SteerSpeed | 漂移恢复 | 行驶油耗 /帧 |
|---|---|---|---|---|---|---|
| **1 = UAZ** | **225** | 250 | 1000 | **150** | 125 | **-0.05** |
| **2 = Volga** | **275** | 250 | 1000 | **125** | — | **-0.07** |

- Volga 更快（275>225）但更费油（0.07>0.05）、转向更钝（125<150）。
- **油尽**（油量≤0）：Accel→0、Decel 增大（2000/250），即**熄火滑行**；停引擎音、播 `car_engine_stop_uaz/volga`、尾气粒子降率（`23453-23533`）。
- **倒车/急停附加减速**：速度 >75/100 且 <-75 时 `Speed -= 125`（或 -50），模拟撞击急停（`23300-23380`）。
- > 油耗 0.05/0.07 是**每帧**（Every-tick）扣减量，非按秒。本项目用 `_physics_process(delta)`，需把"每帧"换算到固定步长（见 §4 实现注）。

### 1.2 载具属性（实例变量 `iv`，`载具数值.md §1`）

| 原版 iv | 含义 | 本项目映射（`DrivableVehicle` 属性） |
|---|---|---|
| `iv[0]` | 车型 (1=UAZ / 2=Volga) | `resource: VehicleResource`（资源本身即车型） |
| `iv[1]` | 当前 HP | `hp: float` |
| `iv[2]` | 最大 HP | `resource.max_hp` |
| `iv[3]` | 当前油量 | `fuel: float` |
| `iv[4]` | 最大油量 | `resource.max_fuel` |
| `iv[5]` | 损毁标志 | `wrecked: bool` |
| 玩家 `iv[44]` 在车 / `iv[45]` 驾驶 UID | 在车/驾驶谁 | 由玩家 ↔ 载具的控制权切换体现（§3） |

> ◻️ **待核实（原版未切净）**：默认 `max_hp`(iv[2])、`max_fuel`(iv[4])、初始 HP/油 的精确数值（`载具数值.md §待核实`：7MB data.js 未精确切片）。
> **暂定占位**（写进 .tres，核实后改）：`max_hp = 150`、`max_fuel = 100`，初始满血满油。出处不足处以注释标明"待核实"。

### 1.3 碰撞伤害（`载具数值.md §3`，`22933-23070`）

- **撞击门槛速度 = 50**：速度 ≤50 不结算伤害/破坏。
- 撞**可破坏障碍**（围栏 `t310` / 木桩 `t305/t306/t560`，cross-ref `02`/`06`）：`HP -= 15`，摧毁该障碍，碎片粒子 + `Fence_destroy`/`car_crash_1~3` 音。
- 撞**硬地物**（`t910`/`t724`）：`HP -= 该物体伤害值`（原版 `物体.iv[0]`），播 `hard_ground` 音。
- > 即：**普通障碍每次有效撞击固定 -15**；硬物按物体自带伤害值。

### 1.4 损毁与重生（`载具数值.md §4`，`23055-23075`）

- `HP ≤ 0` → **爆炸**：播 `car_explode`，火光/烟雾 `t819`、碎片 `t820`、掉落物 `t821`、销毁动画 `t252`。
- **重生计时 = 600 秒**：UAZ→`Uaz_respawn`，Volga→`Volga_respawn`（StartTimer 600s）。被毁 10 分钟后重生。

### 1.5 油料来源（`载具数值.md §5`）

- **加油物品**：Gasoline（原版物品 ID 53）。本项目对应 `item/res/number_item/gasoline.tres`（注：该 .tres 内 `id = 31`、`item_name = "Gasoline"`，与原版 53 编号不同属正常——以**这份资源**为准）。
- **油库 / 油桶**（车旁容器 `t806/t812`）：初始油量随机——大桶 `choose(0, 50)`（`23621/23639`），小桶 `choose(0, 25)`（`23661/23679`）。即 **0 / 25 / 50** 三档。
- > ◻️ Gasoline 每次加注量原版本组未含（归"合成/使用"逻辑），**暂定每次消耗 1 个汽油加满或加固定量**，待核实后调。

### 1.6 车厢供暖（`载具数值.md §1`，cross-ref `08`）

- 玩家**在车内**且**有油**（`fuel>0`）且**未损毁**（`wrecked=false`）→ 体温 **+4**。
- 损毁（`wrecked=true`）→ 不供暖。
- > 本项目用 `08` 定义的 `HeatSource` 契约（同 `10-建造` 的火源做法）注册/注销，不自己另搞一套。

## 2. Godot 节点结构

### 2.1 载具场景 `drivable_vehicle.tscn`（`DrivableVehicle extends CharacterBody2D`）

```
DrivableVehicle  CharacterBody2D  (script drivable_vehicle.gd; collision_layer=2, collision_mask=3)
├── Sprite              Sprite2D            (车体贴图；按 resource.frame 选车型/损坏帧)
├── Body                CollisionShape2D    (近似车身矩形 RectangleShape2D)
├── EnterArea           InteractionArea     (owner_type=OTHER；E 进车；_on_interact→enter())
├── HeatSource          Node2D/Area2D       (08 契约；intact→+4 体温；见 §6)
├── Camera2D            Camera2D            (enabled=false；进车时启用、跟车)
├── ParkMarker          Marker2D            (下车时玩家落点)
└── Effects             Node2D              (引擎尾气/爆炸粒子占位)
```

- **collision_layer=2**：与世界实体层一致（`02 §4` 约定），让别的物体能撞到车。
- **collision_mask=3**（第 1+2 层）：车能撞到建筑/障碍/硬物（与玩家 `mask=3` 同，`player.tscn`）。
- 驾驶时由 `move_and_slide()` 的碰撞结果（`get_slide_collision_count()`）做伤害结算（§5）。

### 2.2 车型资源 `VehicleResource`（`.tres`，每车型一份）

```gdscript

@export var type_id: int = 1            # 1=UAZ, 2=Volga（对应原版 iv[0]）
```

- > 不照搬 Construct2 的 `iv[]`/`t946` 魔法编号——数值落进 `.tres` 字段，逻辑读字段。

## 3. 进 / 出车流程（InteractionArea + 控制权切换 + 相机）

复用 `InteractionArea`（`interact()` 走 `interaction` 输入 = E，门即此用法）。

```
玩家走近车 → EnterArea 检测到玩家 → 按 E → InteractionArea.interact()
  → DrivableVehicle.enter(player):
      1. player.set_physics_process(false) + 隐藏/停靠到 ParkMarker（"泊车"玩家）
      2. 本车接管输入：DrivableVehicle.set_physics_process(true)、driving = true
      3. Camera2D.enabled = true（从玩家相机切到车相机，跟车）
      4. （HeatSource 已常驻；进车不改其状态——供暖看 fuel/wrecked，见 §6）
  → 行驶中再按 E → exit():
      1. driving = false、velocity 归零
      2. 玩家显示在 ParkMarker 全局位置、player.set_physics_process(true)
      3. Camera2D.enabled = false（切回玩家相机）
```

- **控制权切换**而非新建实体：玩家与载具是两个常驻节点，进车只是**交换谁在跑 `_physics_process` + 谁持有 active Camera2D**。
- 玩家相机：若玩家用自身 Camera2D，则进车时 `player_camera.enabled=false` / 车相机启用；出车反之。
- 决策见 §9（D-是否新建实体 / 相机方案）。

## 4. 驾驶控制（镜像 player.gd 的 velocity + move_toward + move_and_slide）

街机俯视转向（accel 到上限、按转向角速度转车头、无漂移）：

```gdscript
func _physics_process(delta):
    if not driving: return
    var throttle := Input.get_axis("move_down", "move_up")   # 前进/倒车
    var steer := Input.get_axis("move_left", "move_right")    # 左右打方向

    # 有油才能加速（原版：仅 fuel>0 可加速）
    if fuel > 0.0 and throttle != 0.0:
        rotation += steer * deg_to_rad(resource.steer_speed) * delta * sign(throttle)
        var target := transform.x * resource.max_speed * throttle   # 车头方向
        velocity = velocity.move_toward(target, resource.accel * delta)
        _consume_fuel(delta)                                          # 行驶耗油
    else:
        # 熄火滑行：油尽或松油门 → 大减速度收敛到 0
        velocity = velocity.move_toward(Vector2.ZERO, resource.decel * delta)

    move_and_slide()
    _resolve_collisions()                                             # §5
```

- **耗油换算**：原版 0.05/0.07 是"每帧"。本项目固定步长 `_physics_process`，按 `fuel -= resource.fuel_per_tick`（每物理帧扣一次）即与原版"Every-tick"对齐；若物理帧率不同于原版，按帧率比缩放（实现时确认 `physics_ticks_per_second`）。
- **油尽**：`fuel<=0` 时不再加速、走 `decel` 滑行分支、播 `engine_stop_sound`、尾气粒子降率（占位）。
- **倒车/急停附加减速**（原版 `23300-23380`）：可选——速度过快反向时额外 `velocity -= 方向*125`，先做基础版，手感再调（见 §9 D-手感）。
- > 无漂移：直接让 `velocity` 朝车头方向收敛（`transform.x`），不做侧向滑移，符合"drift-free"要求。

## 5. 碰撞伤害模型（move_and_slide 碰撞结算）

```gdscript
func _resolve_collisions():
    if velocity.length() <= 50.0: return            # 原版门槛速度 50
    for i in get_slide_collision_count():
        var col := get_slide_collision(i)
        var obj := col.get_collider()
        if obj.is_in_group("destroyable_obstacle"):  # 围栏/木桩，cross-ref 02/06
            take_damage(15.0)
            obj.destroy()                            # 障碍自毁 + 碎片/音效
        elif obj.is_in_group("hard_object"):         # 硬地物 t910/t724
            take_damage(obj.collision_damage)        # 物体自带伤害值
        # 其余实体（建筑等）：仅物理阻挡，不结算（或按需扩展）
```

- 门槛速度 **50**、可破坏障碍 **-15**、硬物 **按物体伤害值**——三条与 `载具数值.md §3` 一致。
- 依赖 `02`/`06` 给可破坏障碍提供 `destroyable_obstacle` 组 + `destroy()`，给硬物提供 `hard_object` 组 + `collision_damage`。本文只**消费**这些约定，不在此定义障碍。

### 损毁 → 爆炸（HP≤0）

```gdscript
func take_damage(amount):
    hp -= amount
    if hp <= 0.0: explode()

func explode():
    wrecked = true                       # 关闭供暖（§6）
    # 播 car_explode、火光/烟雾、掉落物（占位）
    _request_respawn()                   # 通知重生管理器（§7）
    queue_free()                         # 见 §9 D-残骸：是否换成可搜刮静态壳
```

- **决策点（§9 D-残骸）**：被毁的可驾驶载具是
  - (A) `queue_free()` 直接消失，600s 后在槽位重生一辆新车 —— **倾向此方案**（与原版"重生"语义直接对应，且静态车壳残骸已是另一套布景/loot，无需在此复用）；或
  - (B) 原地替换成一个**静态车壳残骸**（`02` 的 `Vehicle`）当 loot，再 600s 重生。

## 6. 车厢供暖（HeatSource，cross-ref 08）

- 用 `08-体温昼夜天气` 定义的 `HeatSource` 契约（与 `10-建造` 火源同款做法）注册到温度系统。
- **供暖条件**：玩家在车内 且 `fuel>0` 且 `wrecked==false` → 提供 **+4** 体温加成。
- 实现：`HeatSource` 节点常驻车上，其"是否生效"由 `driving && fuel>0 && !wrecked` 动态控制（或暴露 `active` 给温度系统查询）。损毁/没油/没人在车→不供暖。
- > 具体 `HeatSource` 字段/注册方式以 `08`（与 `10` 复用的那份契约）为准，本文不另定义接口。

## 7. 重生管理器（600s）

对齐 `01-地图生成` 的"**不按距离 create/erase**"立场——用**简单计时器 + 出生槽位**，不做逐帧距离逻辑。

```
VehicleSpawnManager (Node, 全局/场景级)
├── spawn_slots: Array  # 地图生成时登记的载具出生点 [{pos, type:VehicleResource}]
└── 被毁载具调用 request_respawn(slot)：
      get_tree().create_timer(resource.respawn_seconds).timeout
        → 在该 slot 重新 instantiate 一辆满血满油的车
```

- 出生槽位在**地图/模板生成时**登记（cross-ref `01`/`03`，见 §8），与油库一并落子。
- 用 `SceneTreeTimer`（600s）而非每帧轮询；重生只在 timeout 触发一次，符合 `01` "静态全量 + 不按距离动态生灭"的架构。

## 8. 油库（油料来源世界物体）

简单可交互物体（非可驾驶载具）：

```
OilDepot  StaticBody2D / Area2D
├── Sprite           Sprite2D
├── Shape            CollisionShape2D（可选阻挡）
└── InteractionArea  InteractionArea（E 取油）
```

- 初始储油 `choose(0, 25)` 或 `choose(0, 50)`（按大/小桶），即 **0 / 25 / 50** 三档（`载具数值.md §5`）。
- 交互（E）：把储油注入当前所驾载具的 `fuel`（上限 `max_fuel`），或转成可携带汽油——**倾向直接给车加油**（玩家须开车到油库旁）。细节见 §9 D-油库交互。
- **加油交互（汽油道具）**：玩家持 `gasoline.tres` 在车旁/车内按交互 → 消耗汽油、`fuel = min(fuel+量, max_fuel)`。每次加注量待核实（§1.5）。
- **放置**：可驾驶载具出生槽 + 油库均在地图/模板生成时落子（cross-ref `01`/`03`）。

## 10. 决策

### 已定 ✅
- ✅ **D-类名分离**：可驾驶载具用 `DrivableVehicle`，与既有静态残骸 `Vehicle`(`02`) 彻底分开，互不继承。
- ✅ **D-节点类型**：用 **`CharacterBody2D`**（非 RigidBody2D）。理由：①镜像玩家 `player.gd` 的 `velocity + move_toward + move_and_slide` 已验证的俯视手感，街机式"加速到上限+按角速度转向+无漂移"用运动学体最直接可控；②碰撞伤害靠 `get_slide_collision()` 逐次读取，精确对上原版"门槛速度50 + 固定-15/按伤害值"的离散结算；③RigidBody2D 的真实物理（动量、反弹、扭矩）会引入漂移与不可控弹飞，与原版街机手感和"drift-free"要求相悖，且伤害结算需改用 `body_entered`/接触监测，更绕。
- ✅ **D-重生架构**：简单计时器 + 出生槽位（`SceneTreeTimer` 600s），**不按距离逐帧 create/erase**，对齐 `01` 的"静态全量 + 可见性门控"立场。
- ✅ **D-供暖契约**：复用 `08`（与 `10` 同款）的 `HeatSource`，不另立接口。
